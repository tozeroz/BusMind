# 《智行公交：基于拥挤预测与大模型助手的公交舒适出行系统》项目计划书

## 一、项目简介

本项目拟设计并实现一套面向公交乘客的智能舒适出行系统，项目名称为：

**《智行公交：基于拥挤预测与大模型助手的公交舒适出行系统》**

系统围绕用户“出门乘坐公交”这一具体场景，提供公交线路查询、站点信息查看、车辆预计到站时间 ETA、拥挤程度预测、舒适出行推荐、地图可视化展示和大模型出行助手等功能。

与普通公交查询系统相比，本项目不仅关注“能不能到达”和“多久到达”，还进一步关注“坐得是否舒适”。系统通过对公交线路、站点、车辆位置、历史客流、时段特征等数据进行处理，预测不同线路或车次的拥挤程度，并结合 ETA、步行距离、换乘次数等因素，为用户推荐更适合的公交出行方案。

本项目采用前后端分离架构，前端使用 Vue3 构建用户交互界面，后端使用 FastAPI 提供接口服务，数据库采用 MySQL 存储基础数据和模拟数据，算法模块使用 Python 实现 ETA 计算、拥挤预测和舒适度推荐，大模型模块用于生成自然语言出行建议和路线推荐解释。

------

## 二、项目背景

公交是城市和校园出行中使用频率较高的公共交通方式，具有价格低、覆盖范围广、绿色环保等优点。但在实际出行过程中，乘客仍然面临一些常见问题。

第一，用户往往不知道公交车什么时候到站。虽然部分地图软件可以提供路线规划，但对于具体车辆到站时间、等待时间和车辆运行状态的展示并不一定准确或充分。

第二，用户不知道即将到站的公交是否拥挤。很多情况下，用户到站后才发现车辆已经非常拥挤，乘坐体验较差，甚至可能无法顺利上车。

第三，传统公交查询系统通常以“最快到达”为主要目标，较少考虑用户对舒适度的需求。例如，有些用户愿意多等几分钟，选择一辆不那么拥挤的车；有些用户则更在意少换乘、少步行或准时到达。

第四，普通系统的结果展示较为机械，用户需要自己理解路线、拥挤情况和推荐原因。对于不熟悉线路的用户来说，缺少一个能够用自然语言解释出行方案的智能助手。

因此，本项目希望在传统公交查询功能基础上，引入拥挤程度预测、舒适出行推荐和大模型助手，为用户提供更加智能、个性化、舒适化的公交出行服务。

### 4.2 使用场景

#### 场景一：用户想知道怎么坐公交

用户输入起点和终点，例如“宿舍区到教学楼”，系统返回可选公交方案，包括可乘坐线路、经过站点、预计到站时间和预计到达时间。

用户选择某个站点或某条线路后，系统展示下一班车的预计到站时间 ETA、车辆当前位置、当前运行状态和拥挤程度。

系统综合 ETA、拥挤程度、步行距离和换乘次数，给出“最快方案”“最舒适方案”“最少换乘方案”和“综合推荐方案”。

#### 场景五：用户通过 AI 助手自然语言提问

用户输入“我不想太挤，现在去图书馆怎么坐？”AI 助手结合系统查询结果和预测结果，生成自然语言建议，例如：

> 建议选择校园 2 号线。虽然预计到站时间比校园 1 号线晚约 4 分钟，但当前拥挤程度较低，乘坐体验更好。如果你不赶时间，推荐选择该方案。

## 七、核心算法方案

### 7.1 ETA 到站时间计算

ETA，即 Estimated Time of Arrival，表示预计到站时间。

本项目的 ETA 不强依赖真实公交公司的实时到站接口，而是基于车辆模拟位置、站间距离、平均速度、时段系数、天气系数和拥挤延迟进行计算。

计算公式：

```text
ETA =
剩余站间基础时间 × 时段系数 × 天气系数
+ 站点停靠时间
+ 拥挤延迟
```

其中：

| 因素             | 说明                                     |
| ---------------- | ---------------------------------------- |
| 剩余站间基础时间 | 根据车辆当前位置和目标站点之间的距离计算 |
| 时段系数         | 高峰期车辆速度较慢，因此系数增大         |
| 天气系数         | 雨天、恶劣天气导致速度下降               |
| 站点停靠时间     | 每经过一个站点需要停靠一定时间           |
| 拥挤延迟         | 拥挤车辆上下客时间更长                   |

示例：

```text
车辆距离目标站点还有 3 站；
平均每站需要 3 分钟；
早高峰系数为 1.3；
每站停靠 0.5 分钟。

ETA = 3 × 3 × 1.3 + 3 × 0.5 = 13.2 分钟
```

------

### 7.2 拥挤程度预测

拥挤程度预测是本项目的核心算法功能。

本项目计划采用两阶段实现：

#### 第一阶段：规则模型

先根据时段、站点热度、天气、当前载客率等因素构建规则模型。

```text
拥挤分数 =
当前载客率得分
+ 时段影响得分
+ 站点热度得分
+ 天气影响得分
+ 星期影响得分
```

根据拥挤分数划分为空闲、适中、拥挤、非常拥挤。

#### 第二阶段：机器学习模型

在规则模型基础上，使用模拟历史客流数据训练 XGBoost 或随机森林模型。

输入特征包括：

```text
line_id
station_id
hour
weekday
is_peak
weather
temperature
historical_avg_passenger
current_onboard_count
capacity
```

输出结果：

```text
crowd_level：空闲 / 适中 / 拥挤 / 非常拥挤
```

模型选择建议：

| 模型     | 说明                                                       |
| -------- | ---------------------------------------------------------- |
| 规则模型 | 实现简单，适合作为基础版本                                 |
| 随机森林 | 稳定性较好，适合分类任务                                   |
| XGBoost  | 效果较好，适合表格特征预测                                 |
| LSTM     | 更适合长时间序列，但实现成本较高，短期项目不建议作为主模型 |

本项目建议优先使用 **规则模型 + XGBoost**。规则模型保证系统可运行，XGBoost 作为算法亮点。

------

### 7.3 舒适出行推荐算法

舒适推荐模块综合考虑到站时间、拥挤程度、步行距离和换乘次数，为用户推荐更合适的出行方案。

推荐评分公式：

```text
推荐分数 =
ETA 权重 × ETA 得分
+ 舒适度权重 × 拥挤程度得分
+ 步行距离权重 × 步行得分
+ 换乘次数权重 × 换乘得分
```

默认权重：

```text
score = 0.35 × ETA得分
      + 0.40 × 舒适度得分
      + 0.15 × 步行距离得分
      + 0.10 × 换乘得分
```

不同用户偏好可以调整权重：

| 用户偏好   | 权重调整         |
| ---------- | ---------------- |
| 想最快到达 | 提高 ETA 权重    |
| 不想太挤   | 提高舒适度权重   |
| 不想走太远 | 提高步行距离权重 |
| 不想换乘   | 提高换乘次数权重 |
| 综合推荐   | 各指标均衡       |

推荐结果可以分为：

| 推荐类型     | 说明                                     |
| ------------ | ---------------------------------------- |
| 最快方案     | 总耗时最短                               |
| 最舒适方案   | 拥挤程度最低                             |
| 最少换乘方案 | 换乘次数最少                             |
| 综合推荐方案 | 时间、舒适度、步行距离、换乘次数综合最优 |

------

### 7.4 大模型助手方案

大模型助手主要用于自然语言交互、推荐解释和个性化建议生成。

工作流程：

```text
用户输入自然语言问题
        ↓
系统解析用户起点、终点、时间、偏好
        ↓
调用公交查询、ETA、拥挤预测、舒适推荐结果
        ↓
将结构化结果作为上下文传给大模型
        ↓
大模型生成自然语言出行建议
```

示例输入：

```text
我现在从宿舍区去教学楼，不想太挤，应该坐哪班车？
```

系统结构化数据：

```json
{
  "start": "宿舍区",
  "end": "教学楼",
  "preference": "comfort",
  "routes": [
    {
      "line": "校园1号线",
      "eta": 5,
      "crowd_level": "拥挤",
      "arrival_time": 12
    },
    {
      "line": "校园2号线",
      "eta": 9,
      "crowd_level": "适中",
      "arrival_time": 18
    }
  ]
}
```

大模型输出：

```text
如果你更重视舒适度，建议选择校园2号线。虽然它比校园1号线晚到约6分钟，但当前拥挤程度较低，乘坐体验更好。如果你比较赶时间，可以选择校园1号线，但可能需要站立。
```

## 十四、项目创新点

### 1. 从“公交查询”升级为“舒适出行推荐”

传统公交查询主要关注路线和时间，本项目加入拥挤预测和舒适度评分，能够根据用户偏好推荐更适合的出行方案。

### 2. 将拥挤预测融入公交推荐

系统不仅展示公交 ETA，还预测车内拥挤程度，使用户能够提前判断是否乘坐该车次。

### 3. 引入大模型助手

用户可以使用自然语言提出出行需求，系统结合公交数据、ETA 和拥挤预测结果，生成个性化出行建议。

### 4. 数据来源可控，适合短周期开发

项目使用仿真车辆运行数据和模拟历史客流数据，既避免真实数据获取困难，又能够保证系统逻辑完整。

### 5. 前后端分离，技术栈统一

后端采用 FastAPI，能够同时支持业务接口、算法调用和大模型接入，降低开发复杂度。

## 十七、项目总结定位
