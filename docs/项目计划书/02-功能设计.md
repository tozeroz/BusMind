# 二、功能设计

## 五、项目功能设计

本项目将系统功能划分为六个核心模块：

1. 用户功能模块
2. 公交信息查询
3. 地图展示模块
4. 多源数据融合与出行体验推荐模块
5. 大模型助手模块
6. 数据管理与历史趋势模块

### 5.1 用户功能模块

该模块负责用户基础操作和个性化信息管理。

| 子功能 | 功能说明 |
|---|---|
| 用户注册 | 新用户创建账号 |
| 用户登录 | 用户输入账号和密码登录系统 |
| 个人信息管理 | 用户查看和修改基础信息 |
| 常用路线收藏 | 用户收藏常用线路或常用出行路线 |
| 历史查询记录 | 记录用户曾经查询过的线路、站点和出行方案 |
| 出行偏好设置 | 用户选择综合推荐、最快、舒适优先、少步行、少换乘等偏好 |

------

### 5.2 公交信息查询模块

该模块是用户使用系统的入口，负责展示公交基础信息和地图信息。

| 子功能 | 功能说明 |
|---|---|
| 起终点输入 | 用户输入出发地和目的地 |
| 公交线路查询 | 根据起终点查询可选公交线路 |
| 站点列表展示 | 展示某条线路经过的全部站点 |
| 线路详情查看 | 展示首末班时间、运行方向、站点顺序等信息 |
| 车辆位置展示 | 在地图上展示车辆当前位置或模拟位置 |
| ETA 到站时间展示 | 展示车辆到达目标站点的实时预计时间 |
| 客载状态展示 | 展示车辆当前客载状态，例如有座、可站立、较拥挤 |
| 地图可视化 | 展示线路走向、站点位置、车辆位置和推荐路线 |

### 5.3 地图可视化模块

地图可视化模块用于增强系统展示效果，让用户能够直观看到公交线路、车辆位置、客载状态和道路热力。

| 子功能 | 功能说明 | 优先级 |
|---|---|---|
| 车辆位置 | 在地图上展示公交车辆当前位置或模拟运行位置 | P1 |
| 路线方案展示 | 在地图上展示推荐路线、经过站点和行驶方向 | P1 |
| 客载状态展示 | 通过颜色或标识展示线路、站点或车辆的客载状态 | P1 |
| 道路热力展示 | 展示途经道路拥堵程度或速度热力 | P1 |
| 站点客流展示 | 展示站点历史客流压力和时段变化 | P2 |

### 5.4 多源数据融合与出行体验推荐模块

该模块是项目核心模块，负责实时公交状态提取、历史客流分析、道路热力分析、候选路线生成和出行体验评价与推荐。

| 子功能 | 功能说明 |
|---|---|
| 实时公交状态提取 | 从 Bus Arrival 获取 `eta_minutes`、`Load`、车辆位置和数据可信度 |
| 候选路线生成 | 根据起终点匹配附近站点，生成直达或一次换乘公交方案 |
| 历史客流分析 | 根据站点小时级 tap in / tap out 计算客流压力和 `history_flow_score` |
| 道路热力分析 | 根据 Traffic Speed Bands 计算道路拥堵压力和 `congestion_score` |
| 出行体验评分 | 综合时间、舒适、步行、换乘、可靠性计算分项得分 |
| 个性化路线推荐 | 根据用户偏好动态调整权重，输出 Top-K 推荐方案 |
| 推荐理由解释 | 生成结构化推荐原因，供页面和 AI 助手使用 |

客载状态使用 LTA DataMall `Load` 字段的三类标签：

| Load 标签 | 含义 | 项目展示 | load_score |
|---|---|---|---:|
| SEA | Seats Available | 有座位，较宽松 | 100 |
| SDA | Standing Available | 可站立，适中 | 70 |
| LSD | Limited Standing | 站立空间有限，较拥挤 | 35 |

推荐模块输出以下推荐类型：

| 推荐类型 | 展示名称 | 选择规则 |
|---|---|---|
| `best_experience` | 综合推荐 | 当前用户偏好下 `recommend_score` 最高 |
| `fastest` | 最快到达 | `total_time_minutes` 最小 |
| `least_crowded` | 车内更宽松 | `comfort_score` 最高 |
| `least_transfer` | 换乘更少 | `transfer_count` 最少，相同则推荐分更高 |
| `least_walking` | 步行更少 | `walk_time_minutes` 最短，相同则推荐分更高 |
| `most_reliable` | 数据更可靠 | `reliability_score` 最高 |

### 5.5 大模型助手模块

该模块负责自然语言交互和智能解释，是项目的重要创新点。

| 子功能 | 功能说明 |
|---|---|
| 自然语言问答 | 用户可以用自然语言询问公交出行问题 |
| 个性化出行建议 | 根据用户偏好生成路线建议 |
| 路线推荐解释 | 解释某条路线为什么更适合用户 |
| 出行提醒生成 | 生成错峰出行、提前出门、换乘提醒等提示 |
| 公交知识问答 | 回答线路、站点、换乘等相关问题 |

大模型助手不直接负责路线计算，不计算 ETA，不预测 Load，不修改推荐分数，而是调用系统已有的候选路线、实时状态、客流热力和推荐结果，再生成用户容易理解的自然语言回答。

------

### 5.6 数据管理与历史趋势模块

该模块为推荐和可视化提供数据支撑。

| 子功能 | 功能说明 |
|---|---|
| 线路数据管理 | 维护公交线路基础信息 |
| 站点数据管理 | 维护站点名称、经纬度和站点顺序 |
| 车辆实时数据管理 | 管理车辆位置、ETA、Load 和监控状态等实时或模拟数据 |
| 历史客流趋势查看 | 查看某线路或站点在不同时间段的客流压力变化 |
| 道路热力数据管理 | 存储或模拟 Traffic Speed Bands，支持拥堵得分计算 |
| 推荐结果记录 | 保存分项得分、综合推荐分和推荐理由，方便展示和分析 |
