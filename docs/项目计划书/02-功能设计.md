# 二、功能设计

## 五、项目功能设计

本项目将系统功能划分为六个核心模块：

1. 用户功能模块
2. 公交信息查询
3. 地图展示模块
4. 智能预测与出行体验推荐模块
5. 大模型助手模块
6. 数据管理与历史趋势模块

### 5.1 用户功能模块

该模块负责用户基础操作和个性化信息管理。

| 子功能       | 功能说明                                 |
| ------------ | ---------------------------------------- |
| 用户注册     | 新用户创建账号                           |
| 用户登录     | 用户输入账号和密码登录系统               |
| 个人信息管理 | 用户查看和修改基础信息                   |
| 常用路线收藏 | 用户收藏常用线路或常用出行路线           |
| 历史查询记录 | 记录用户曾经查询过的线路、站点和出行方案 |

------

### 5.2 公交信息查询模块

该模块是用户使用系统的入口，负责展示公交基础信息和地图信息。

| 子功能           | 功能说明                                   |
| ---------------- | ------------------------------------------ |
| 起终点输入       | 用户输入出发地和目的地                     |
| 公交线路查询     | 根据起终点查询可选公交线路                 |
| 站点列表展示     | 展示某条线路经过的全部站点                 |
| 线路详情查看     | 展示首末班时间、运行方向、站点顺序等信息   |
| 车辆位置展示     | 在地图上展示车辆当前位置或模拟位置         |
| ETA 到站时间展示 | 展示车辆到达目标站点的预计时间             |
| 运行状态展示     | 展示车辆正常、延误、拥挤等状态             |
| 地图可视化       | 展示线路走向、站点位置、车辆位置和推荐路线 |

### 5.3 地图可视化模块

地图可视化模块用于增强系统展示效果，让用户能够直观看到公交线路、车辆位置和客载状态。

| 子功能       | 功能说明                                     | 优先级 |
| ------------ | -------------------------------------------- | ------ |
| 车辆位置     | 在地图上展示公交车辆当前位置或模拟运行位置   | P1     |
| 路线方案展示 | 在地图上展示推荐路线、经过站点和行驶方向     | P1     |
| 客载状态展示 | 通过颜色或标识展示线路、站点或车辆的客载状态 | P1     |

------

### 5.4 智能预测与出行体验推荐模块

该模块是项目核心模块，负责 ETA 预测、客载状态预测和出行体验评价与推荐。

| 子功能           | 功能说明                                                     |
| ---------------- | ------------------------------------------------------------ |
| ETA 预测         | 根据线路、站点、车辆位置、时段、天气、历史 ETA 等特征预测到站时间 |
| 客载状态预测     | 根据线路、站点、时段、客流、OD、天气和历史 Load 预测车内客载状态 |
| 出行体验评分     | 综合预测 ETA、预测客载状态、步行时间和换乘次数计算方案得分   |
| 出行体验推荐     | 推荐最快、车内更宽松、最少换乘、综合最优方案                 |
| 推荐理由解释     | 解释系统为什么推荐该线路或车次                               |

客载状态使用 LTA DataMall `Load` 字段的三类标签：

| Load 标签 | 含义 | 项目展示 |
| --------- | ---- | -------- |
| SEA       | Seats Available | 有座位，较宽松 |
| SDA       | Standing Available | 可站立，适中 |
| LSD       | Limited Standing | 站立空间有限，较拥挤 |

------

### 5.5 大模型助手模块

该模块负责自然语言交互和智能解释，是项目的重要创新点。

| 子功能         | 功能说明                               |
| -------------- | -------------------------------------- |
| 自然语言问答   | 用户可以用自然语言询问公交出行问题     |
| 个性化出行建议 | 根据用户偏好生成路线建议               |
| 路线推荐解释   | 解释某条路线为什么更适合用户           |
| 出行提醒生成   | 生成错峰出行、提前出门、换乘提醒等提示 |
| 公交知识问答   | 回答线路、站点、换乘等相关问题         |

大模型助手不直接负责路线计算，而是调用系统已有的线路、ETA 预测、客载状态预测和推荐结果，再生成用户容易理解的自然语言回答。

------

### 5.6 数据管理与历史趋势模块

该模块为预测、推荐和可视化提供数据支撑。

| 子功能           | 功能说明                                   |
| ---------------- | ------------------------------------------ |
| 线路数据管理     | 维护公交线路基础信息                       |
| 站点数据管理     | 维护站点名称、经纬度和站点顺序             |
| 车辆模拟数据管理 | 管理车辆位置、速度、客载状态等模拟数据     |
| 历史客载趋势查看 | 查看某线路或站点在不同时间段的客载状态变化 |
| 客流数据统计     | 展示不同时段、不同站点的客流情况           |
| 预测结果记录     | 保存 ETA 预测、客载状态预测和推荐结果，方便展示和分析 |

------
