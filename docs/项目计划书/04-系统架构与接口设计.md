# 四、系统架构与接口设计

## 八、系统总体架构

本项目采用前后端分离架构，整体架构如下：

```text
用户浏览器
   ↓
Vue3 前端页面
   ↓ Axios 请求
FastAPI 后端服务
   ↓
业务服务层
   ├── 用户服务
   ├── 公交查询服务
   ├── ETA 预测服务
   ├── 客载状态预测服务
   ├── 出行体验推荐服务
   └── AI 助手服务
   ↓
数据访问层 SQLAlchemy
   ↓
MySQL 数据库

FastAPI 后端
   ↓
Python 算法模块
   ├── 数据处理
   ├── ETA 预测
   ├── 客载状态预测
   └── 出行体验评分

FastAPI 后端
   ↓
大模型 API
   ↓
自然语言出行建议
```

核心数据流：

```text
线路和站点查询
        ↓
生成候选公交方案
        ↓
调用 ETA 预测模型，得到 predicted_eta_minutes
        ↓
调用客载状态预测模型，得到 predicted_load 和 load_score
        ↓
调用推荐模块，计算 experience_score
        ↓
返回推荐方案，并交给 AI 助手生成解释
```

## 九、技术选型

| 模块 | 技术选型 | 说明 |
|---|---|---|
| 前端框架 | Vue3 | 构建用户交互页面 |
| UI 组件 | Element Plus | 快速开发表单、表格、卡片等页面组件 |
| 地图展示 | 高德地图 API / 百度地图 API | 展示线路、站点、车辆位置 |
| 图表展示 | ECharts | 展示客流趋势、客载状态分布等图表 |
| 后端框架 | FastAPI | 提供 RESTful API，自动生成接口文档 |
| 数据库 | MySQL | 存储用户、线路、站点、车辆、历史客流、预测结果等数据 |
| ORM | SQLAlchemy | 操作数据库 |
| 数据处理 | Python + Pandas + NumPy | 处理 LTA 风格数据和仿真数据 |
| 机器学习 | Scikit-learn / XGBoost | 实现 ETA 回归预测和 Load 三分类预测 |
| 大模型接口 | DeepSeek / 通义千问 / OpenAI API | 实现 AI 出行助手 |
| 接口通信 | Axios + RESTful API | 前后端数据交互 |
| 接口测试 | Apifox / Postman | 接口调试和文档管理 |
| 项目管理 | Git / GitHub / Gitee | 代码版本管理 |
| 部署方式 | 本地部署 / 云服务器 + Nginx | 项目演示和答辩部署 |

### 11.2 公交信息查询接口

| 接口 | 方法 | 说明 |
|---|---|---|
| `/api/bus/lines` | GET | 查询所有公交线路 |

| `/api/history/load` | GET | 查询历史客载状态趋势 |

## 十二、数据库设计初稿

建议至少设计以下数据表：

| 数据表 | 作用 |
|---|---|
| `user` | 存储用户信息 |
| `bus_line` | 存储公交线路信息 |
| `bus_station` | 存储站点信息 |
| `line_station` | 存储线路与站点的关联关系 |
| `bus_vehicle` | 存储车辆基础信息 |
| `bus_realtime` | 存储车辆实时位置和运行状态 |
| `lta_bus_arrival` | 存储或模拟 `EstimatedArrival`、车辆位置、`Load` 等采样数据 |
| `passenger_volume_stop` | 存储站点小时级 tap in / tap out 客流 |
| `passenger_volume_od` | 存储 OD 站点对小时级出行量 |
| `eta_prediction` | 存储 ETA 预测结果和评价信息 |
| `load_prediction` | 存储客载状态预测结果和评价信息 |
| `route_recommendation` | 存储推荐方案、体验分和推荐理由 |
| `user_favorite` | 存储用户收藏路线 |
| `query_history` | 存储用户查询记录 |

第一阶段数据库重点保证以下链路可用：

```text
线路站点基础数据
        ↓
LTA 风格到站和客载状态样本
        ↓
ETA 预测结果
        ↓
客载状态预测结果
        ↓
出行体验推荐结果
```
