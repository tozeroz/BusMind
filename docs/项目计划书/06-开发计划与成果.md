# 六、开发计划与成果

## 十五、开发计划

| 时间 | 任务 |
|---|---|
| 第 1 天 | 明确需求，确定功能模块，完成项目计划书 |
| 第 2 天 | 设计数据库表结构，完成接口文档初稿 |
| 第 3 天 | 搭建 Vue3 前端和 FastAPI 后端基础框架 |
| 第 4 天 | 完成用户登录注册、线路查询、站点查询功能 |
| 第 5 天 | 完成 Bus Stops、Bus Routes、Bus Services 和 Bus Arrival 数据导入或仿真 |
| 第 6 天 | 完成候选路线生成、实时 ETA / Load 解析和历史客流压力计算 |
| 第 7 天 | 完成道路热力分析、多目标评分、个性化推荐和推荐结果页面 |
| 第 8 天 | 接入大模型助手，实现结构化推荐解释和自然语言出行建议 |
| 第 9 天 | 前后端联调，接口测试，补充缺失数据降级策略 |
| 第 10 天 | 整理项目文档、制作 PPT、准备答辩演示 |

## 十七、项目总结定位

本项目最终定位为：

> 面向公交乘客的智能出行体验推荐系统。系统通过公交信息查询、实时 ETA 与客载状态展示、历史客流和道路热力融合、多目标出行体验推荐和大模型助手，为用户提供更加舒适、智能、个性化的公交出行服务。

一句话概括就是：

> **不只是告诉用户怎么坐公交，而是告诉用户怎么坐得更舒服。**
