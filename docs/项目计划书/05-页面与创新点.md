# 五、页面与创新点

## 十三、页面设计方案

| 页面 | 页面内容 |
|---|---|
| 登录注册页 | 用户登录、注册 |
| 首页 | 起终点输入、出行偏好选择、常用线路、AI 助手入口 |
| 线路查询页 | 展示可选线路、站点列表、首末班时间 |
| 公交详情页 | 地图展示车辆位置、实时 ETA、实时客载状态 |
| 出行体验推荐页 | 展示综合推荐、最快到达、车内更宽松、换乘更少、步行更少等方案 |
| 推荐详情页 | 展示时间、舒适、步行、换乘、可靠性等分项得分和推荐原因 |
| AI 出行助手页 | 聊天式问答，生成自然语言出行建议 |
| 历史趋势页 | 查看线路或站点历史客流、客载状态和道路热力变化 |

## 十四、项目创新点

### 1. 从“公交查询”升级为“出行体验推荐”

传统公交查询主要关注路线和时间，本项目进一步加入实时客载、历史客流、道路热力、步行负担、换乘复杂度和数据可靠性，能够根据用户偏好推荐更适合的出行方案。

### 2. 多源数据融合，而不是单一最快路线

系统将 LTA Bus Arrival、Bus Stops、Bus Routes、Bus Services、Passenger Volume by Bus Stops 和 Traffic Speed Bands 等数据统一整理为推荐特征，让实时状态、历史趋势和道路环境共同参与路线排序。

### 3. 多目标评分与动态偏好权重

推荐算法不只输出最快路线，而是计算 time、comfort、walk、transfer、reliability 等分项得分，并根据综合推荐、最快、舒适优先、少步行、少换乘等用户偏好动态调整权重。

### 4. 可解释推荐结果

系统输出推荐分数的同时生成结构化推荐原因，例如 ETA 较短、客载较宽松、历史客流压力较低、道路拥堵较轻、无需换乘等，方便前端展示和 AI 助手解释。

### 5. 大模型助手只做解释，不替代核心算法

大模型读取系统已经计算好的结构化推荐结果，负责把路线方案转化为自然语言建议。它不直接计算 ETA，不预测 Load，不修改推荐分数，保证推荐逻辑清晰可控。

### 6. 数据方案可替换，适合短周期开发

项目参考 LTA DataMall 的 `Bus Arrival`、`Bus Routes`、`Bus Stops`、`Bus Services`、站点客流和道路速度热力数据结构。真实数据不足时可按同字段生成仿真数据，既保证短周期可落地，又便于后续替换真实数据源。

### 7. 前后端分离，技术栈统一

后端采用 FastAPI，能够同时支持业务接口、实时状态接口、推荐算法调用和大模型接入，降低开发复杂度。
