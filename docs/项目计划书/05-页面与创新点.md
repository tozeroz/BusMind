# 五、页面与创新点

## 十三、页面设计方案

| 页面          | 页面内容                           |
| ------------- | ---------------------------------- |
| 登录注册页    | 用户登录、注册                     |
| 首页          | 起终点输入、常用线路、AI 助手入口  |
| 线路查询页    | 展示可选线路、站点列表、首末班时间 |

| 舒适推荐页    | 展示最快、最舒适、综合推荐方案     |
| AI 出行助手页 | 聊天式问答，生成自然语言出行建议   |
| 历史趋势页    | 查看线路或站点历史拥挤变化         |

## 十四、项目创新点

### 1. 从“公交查询”升级为“舒适出行推荐”

传统公交查询主要关注路线和时间，本项目加入客载量预测和出行体验评价指标评分，能够根据用户偏好推荐更适合的出行方案。

### 2. 将客载量预测融入公交推荐

系统不仅展示公交 ETA，还预测车内拥挤程度，使用户能够提前判断是否乘坐该车次。

### 3. 引入大模型助手

用户可以使用自然语言提出出行需求，系统结合公交数据、ETA 和客载量预测结果，生成个性化出行建议。

### 4. 数据来源可控，适合短周期开发

项目使用仿真车辆运行数据和模拟历史客流数据，既避免真实数据获取困难，又能够保证系统逻辑完整。

### 5. 前后端分离，技术栈统一

后端采用 FastAPI，能够同时支持业务接口、算法调用和大模型接入，降低开发复杂度。
