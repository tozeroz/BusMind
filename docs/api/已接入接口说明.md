# BusMind 已封装接口说明

> 本文用于区分 **已封装（前端 axios 函数）** 与 **已实际接入（页面触发）** 两种范围：
> - 第 4 节枚举的是 `frontend/src/api/*` 中**已封装**的接口方法（即前端可调用入口）。
> - 第 5 节枚举的是前端页面**真正会发出去**的请求（即演示链路上可观察到的请求）。
>
> 如果后续在文档中看到 "已接入" 等措辞，请统一理解为 "已封装"。

## 1. 文档依据

本文档按当前后端 FastAPI OpenAPI 定义整理。当前 `127.0.0.1:8000` 未成功监听时，可通过 FastAPI 应用对象生成同一份 OpenAPI：

## 2. 基础约定

| 项目 | 说明 |
|---|---|
| 后端服务地址 | `http://127.0.0.1:8000` |
| 统一接口前缀 | `/api/v1` |
| 前端开发代理 | Vite 将 `/api` 代理到 `http://127.0.0.1:8000` |
| 前端 Base URL | `VITE_API_BASE_URL=/api/v1` |
| 请求格式 | `GET` 使用 Query/Path；`POST`、`PATCH` 使用 `application/json` |
| 响应格式 | 统一返回 `code`、`message`、`data`、`trace_id`、`timestamp` |
| 鉴权方式 | 需要登录的接口使用 `Authorization: Bearer <access_token>` |

统一响应示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "trace_id": "req_xxx",
  "timestamp": "2026-07-10T00:00:00+08:00"
}
```

前端 Axios 封装位置：

```text
frontend/src/api/request.js
```

该文件会自动读取 `localStorage` 或 `sessionStorage` 中的 `busmind_access_token` / `access_token`，并添加 Bearer Token。

## 5. 前端调用规范

页面组件中只调用 `frontend/src/api` 中的函数，不直接拼接完整后端地址。

推荐写法：

```js
import { getLines, getRealtimeVehicles, recommendRoutes } from '@/api'

const lineResponse = await getLines({ page: 1, limit: 20 })
const vehicleResponse = await getRealtimeVehicles({ line_id: 1 })
const recommendResponse = await recommendRoutes({
  start_station_id: 1,
  end_station_id: 5,
  preference: 'balanced'
})
```

不推荐写法：

```js
axios.get('http://127.0.0.1:8000/api/v1/lines')
```

## 6. 当前路径差异说明

| 早期文档路径 | 当前 OpenAPI 实际情况 | 当前限制 |
|---|---|---|
| `/api/v1/bus-lines` | 已提供 GET 兼容别名；完整增删改使用 `/api/v1/lines` | 无 |
| `/api/v1/bus-lines/{line_id}/map` | **已封装**；返回线路的 polyline、stations、bounds | `direction` 参数预留，暂未实现方向过滤 |
| `/api/v1/bus-stations/nearby` | **已封装** GET 版本；同时保留 `POST /api/v1/stations/nearby` | 无 |
| `/api/v1/bus-vehicles/realtime` | **已封装**；兼容别名，等效于 `/api/v1/vehicles/realtime` | 无 |
| `/api/v1/history/passenger-load` | **已封装**；兼容接口，复用 `/api/v1/history/load/...` 逻辑 | 最小实现，参数有限 |
| `/api/v1/history/predictions` | **已封装**；聚合接口，统一返回 ETA、客载、客流预测 | 最小实现，无分页 |

## 7. 变更总结（2026-07-10）

### 7.1 新增接口

| 接口路径 | 类型 | 说明 |
|---|---|---|
| `GET /api/v1/bus-vehicles/realtime` | 兼容别名 | 等效于 `/api/v1/vehicles/realtime` |
| `GET /api/v1/bus-stations/nearby` | 新接口 | GET 方式查询附近站点，补充原有 POST 版本 |
| `GET /api/v1/bus-lines/{line_id}/map` | 新接口 | 返回单条线路的地图数据（polyline、stations、bounds） |
| `GET /api/v1/history/passenger-load` | 兼容接口 | 兼容文档路径，复用 `/api/v1/history/load/...` 逻辑 |
| `GET /api/v1/history/predictions` | 聚合接口 | 统一返回 ETA、客载、客流预测数据 |

### 7.2 当前限制

- `GET /api/v1/bus-lines/{line_id}/map` 的 `direction` 参数为预留字段，当前未实现方向过滤功能
- `GET /api/v1/history/passenger-load` 为最小实现，参数有限（仅支持 line_id、station_id、vehicle_id）
- `GET /api/v1/history/predictions` 为最小实现，无分页和时间范围过滤功能

### 7.3 接口真实化状态

| 接口路径 | 状态 | 说明 |
|---|---|---|
| `/api/v1/vehicles/realtime` | 真实可用 | 支持真实线路 ID 查询 |
| `/api/v1/location/nearby` | 真实可用 | 支持真实坐标查询附近站点 |
| `/api/v1/eta` | 真实可用 | 支持真实车辆和站点 ID |
| `/api/v1/passenger-load-prediction` | 真实可用 | 支持真实线路和站点 ID |
| `/api/v1/walking-time-estimation` | 最小真实版 | 支持真实站点 ID，使用直线距离估算 |
| `/api/v1/recommend-routes` | 最小真实版 | 支持真实站点 ID，优先直达路线 |
| `/api/v1/history/passenger-load` | 兼容版 | 复用现有 load 查询逻辑 |
| `/api/v1/history/predictions` | 聚合版 | 统一返回多种预测类型 |

### 7.4 接口对比

| 接口 | 用途 |
|---|---|
| `/api/v1/map/lines` | 获取所有线路的概览数据 |
| `/api/v1/bus-lines/{line_id}/map` | 获取单条线路的详细地图数据（含站点列表和视野边界） |

### 8.4 调用链路示例

**首页地图加载真实站点与轨迹**（`BusMap.vue` → `map.on('load')`）：

```text
GET /api/v1/lines?page=1&limit=10
        ↓ 过滤 Postman / {{ 占位，取前 9 条
GET /api/v1/map/stations?line_id={id} × 9（并行）
        ↓ 合并去重 → busStopsGeoJSON
GET /api/v1/map/lines
        ↓ 按 visibleLineIds 过滤 → 转 LineString GeoJSON
MapLibre routes 数据源 setData
```

**线路详情**（`LineDetailView.vue` → `onMounted`）：

```text
GET /api/v1/lines/{line_id}
        ↓
渲染线路信息表 + 真实站点顺序
```

**AI 出行问答**（`HomeView.vue` / `AiAssistantView.vue` → 提交输入）：

```text
POST /api/v1/ai/travel  body: { mode, question, context }
        ↓
渲染 AI 卡片答复
```

**后台 LTA 刷新**（`AdminView.vue` → 点击刷新）：

```text
POST /api/v1/admin/lta/bus-arrival/refresh  body: { bus_stop_code, service_no, sync_to_db }
        ↓
展示同步结果
```
