# BusMind 已接入接口说明

## 1. 文档依据

本文档按当前后端 FastAPI OpenAPI 定义整理。当前 `127.0.0.1:8000` 未成功监听时，可通过 FastAPI 应用对象生成同一份 OpenAPI：

```powershell
cd E:\大学\26小学期\code\backend
$env:PYTHONPATH='E:\大学\26小学期\code'
python -c "from app.main import app; print(app.openapi()['paths'].keys())"
```

补充参考：

```text
docs/api/BusMind项目接口文档.md
```

如两者不一致，以当前 OpenAPI 为准。

## 2. 基础约定

| 项目 | 说明 |
|---|---|
| 后端服务地址 | `http://127.0.0.1:8000` |
| 统一接口前缀 | `/api/v1` |
| 前端开发代理 | Vite 将 `/api` 代理到 `http://127.0.0.1:8000` |
| 前端 Base URL | `VITE_API_BASE_URL=/api/v1` |
| 请求格式 | `GET` 使用 Query/Path；`POST`、`PATCH` 使用 `application/json` |
| 响应格式 | 统一返回 `code`、`message`、`data`、`trace_id`、`timestamp` |
| 鉴权方式 | 需要登录的接口使用 `Authorization: Bearer <access_token>` |

统一响应示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "trace_id": "req_xxx",
  "timestamp": "2026-07-10T00:00:00+08:00"
}
```

前端 Axios 封装位置：

```text
frontend/src/api/request.js
```

该文件会自动读取 `localStorage` 或 `sessionStorage` 中的 `busmind_access_token` / `access_token`，并添加 Bearer Token。

## 5. 前端调用规范

页面组件中只调用 `frontend/src/api` 中的函数，不直接拼接完整后端地址。

推荐写法：

```js
import { getLines, getRealtimeVehicles, recommendRoutes } from '@/api'

const lineResponse = await getLines({ page: 1, limit: 20 })
const vehicleResponse = await getRealtimeVehicles({ line_id: 1 })
const recommendResponse = await recommendRoutes({
  start_station_id: 1,
  end_station_id: 5,
  preference: 'balanced'
})
```

不推荐写法：

```js
axios.get('http://127.0.0.1:8000/api/v1/lines')
```

## 6. 当前路径差异说明

| 早期文档路径 | 当前 OpenAPI 实际情况 | 当前限制 |
|---|---|---|
| `/api/v1/bus-lines` | 已提供 GET 兼容别名；完整增删改使用 `/api/v1/lines` | 无 |
| `/api/v1/bus-lines/{line_id}/map` | **已接入**；返回线路的 polyline、stations、bounds | `direction` 参数预留，暂未实现方向过滤 |
| `/api/v1/bus-stations/nearby` | **已接入** GET 版本；同时保留 `POST /api/v1/stations/nearby` | 无 |
| `/api/v1/bus-vehicles/realtime` | **已接入**；兼容别名，等效于 `/api/v1/vehicles/realtime` | 无 |
| `/api/v1/history/passenger-load` | 当前未暴露；实际为 `/api/v1/history/load/...` | - |
| `/api/v1/history/predictions` | 当前未暴露；实际拆成 ETA 与 load 相关路径 | - |

## 7. 变更总结（2026-07-10）

### 7.1 新增接口

| 接口路径 | 类型 | 说明 |
|---|---|---|
| `GET /api/v1/bus-vehicles/realtime` | 兼容别名 | 等效于 `/api/v1/vehicles/realtime` |
| `GET /api/v1/bus-stations/nearby` | 新接口 | GET 方式查询附近站点，补充原有 POST 版本 |
| `GET /api/v1/bus-lines/{line_id}/map` | 新接口 | 返回单条线路的地图数据（polyline、stations、bounds） |

### 7.2 当前限制

- `GET /api/v1/bus-lines/{line_id}/map` 的 `direction` 参数为预留字段，当前未实现方向过滤功能

### 7.3 接口对比

| 接口 | 用途 |
|---|---|
| `/api/v1/map/lines` | 获取所有线路的概览数据 |
| `/api/v1/bus-lines/{line_id}/map` | 获取单条线路的详细地图数据（含站点列表和视野边界） |
