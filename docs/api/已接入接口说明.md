# BusMind 已接入接口说明

## 1. 文档依据

本文档按当前后端实际暴露的 OpenAPI 文档整理，读取来源为：

```text
http://127.0.0.1:8000/openapi.json
```

补充参考：

```text
docs/api/BusMind项目接口文档.md
```

如两者不一致，以当前 OpenAPI 为准。

## 2. 基础约定

| 项目 | 说明 |
|---|---|
| 后端服务地址 | `http://127.0.0.1:8000` |
| 统一接口前缀 | `/api/v1` |
| 前端开发代理 | Vite 将 `/api` 代理到 `http://127.0.0.1:8000` |
| 前端 Base URL | `VITE_API_BASE_URL=/api/v1` |
| 请求格式 | `GET` 使用 Query/Path；`POST`、`PATCH` 使用 `application/json` |
| 响应格式 | 统一返回 `code`、`message`、`data`、`trace_id`、`timestamp` |
| 鉴权方式 | 需要登录的接口使用 `Authorization: Bearer <access_token>` |

统一响应示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "trace_id": "req_xxx",
  "timestamp": "2026-07-09T18:00:00+08:00"
}
```

前端 Axios 封装已接入：

```text
frontend/src/api/request.js
```

该文件会自动读取 `localStorage` 或 `sessionStorage` 中的 `busmind_access_token` / `access_token`，并添加 Bearer Token。

## 3. 前端接口文件

| 文件 | 说明 |
|---|---|
| `frontend/src/api/request.js` | Axios 实例、统一 `baseURL`、请求鉴权头、响应解包 |
| `frontend/src/api/user.js` | 用户注册、登录、当前用户信息 |
| `frontend/src/api/intelligence.js` | ETA、客载预测、步行估算、体验评价、路线推荐 |
| `frontend/src/api/bus.js` | 公交智能接口兼容导出，保留原函数名 |
| `frontend/src/api/ai.js` | AI 出行助手 |
| `frontend/src/api/simulation.js` | 车辆状态、预测结果、LTA 到站刷新等仿真更新接口 |
| `frontend/src/api/index.js` | 统一导出所有接口方法 |

## 4. 已接入接口清单

### 4.1 用户模块

| 接口名称 | 方法 | 路径 | 前端方法 | 鉴权 |
|---|---|---|---|---|
| 用户注册 | `POST` | `/api/v1/users/register` | `registerUser(data)` | 否 |
| 用户登录 | `POST` | `/api/v1/users/login` | `loginUser(data)` | 否 |
| 获取当前用户 | `GET` | `/api/v1/users/me` | `getCurrentUser()` | 是 |
| 修改当前用户 | `PATCH` | `/api/v1/users/me` | `updateCurrentUser(data)` | 是 |

注册请求：

```json
{
  "username": "test_user",
  "password": "12345678",
  "nickname": "测试用户"
}
```

登录请求：

```json
{
  "username": "test_user",
  "password": "12345678"
}
```

登录成功后，前端可保存令牌：

```js
import { loginUser, saveAuthToken } from '@/api/user'

const response = await loginUser({ username, password })
saveAuthToken(response.data.access_token)
```

### 4.2 ETA 模块

| 接口名称 | 方法 | 路径 | 前端方法 | 鉴权 |
|---|---|---|---|---|
| 查询预计到站时间 | `GET` | `/api/v1/eta` | `getEta(params)` | 否 |

Query 参数：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `vehicle_id` | int | 是 | 车辆编号 |
| `target_station_id` | int | 是 | 目标站点编号 |
| `line_id` | int | 否 | 线路编号 |
| `query_time` | datetime | 否 | 查询时间 |

前端调用：

```js
import { getEta } from '@/api/bus'

const response = await getEta({
  vehicle_id: 1,
  target_station_id: 3,
  line_id: 1
})
```

### 4.3 客载预测模块

| 接口名称 | 方法 | 路径 | 前端方法 | 鉴权 |
|---|---|---|---|---|
| 预测车辆客载状态 | `POST` | `/api/v1/passenger-load-prediction` | `predictPassengerLoad(data)` | 否 |

请求体：

```json
{
  "line_id": 1,
  "station_id": 2,
  "vehicle_id": 10,
  "target_time": "2026-07-09T18:00:00+08:00",
  "current_onboard_count": 32,
  "capacity": 60,
  "weather": "clear"
}
```

其中 `line_id`、`station_id` 必填，其余字段可选。

### 4.4 步行估算模块

| 接口名称 | 方法 | 路径 | 前端方法 | 鉴权 |
|---|---|---|---|---|
| 估算到上车站步行时间 | `POST` | `/api/v1/walking-time-estimation` | `estimateWalkingTime(data)` | 否 |

请求体：

```json
{
  "origin_longitude": 116.397,
  "origin_latitude": 39.908,
  "target_station_id": 2,
  "walking_speed_mps": 1.2,
  "route_mode": "straight_line"
}
```

`route_mode` 可选值：

```text
straight_line / map_api
```

### 4.5 体验评价与路线推荐模块

| 接口名称 | 方法 | 路径 | 前端方法 | 鉴权 |
|---|---|---|---|---|
| 计算出行体验分 | `POST` | `/api/v1/travel-experience/evaluate` | `evaluateTravelExperience(data)` | 否 |
| 生成路线推荐 | `POST` | `/api/v1/recommend-routes` | `recommendRoutes(data)` | 否 |

体验评价请求体：

```json
{
  "predicted_load_rate": 0.75,
  "predicted_load_level": "standing_available",
  "transfer_count": 1,
  "walk_time_minutes": 6,
  "weights": {
    "w_load": 0.5,
    "w_walk": 0.3,
    "w_transfer": 0.2
  }
}
```

路线推荐请求体支持两种起终点形式。

站点编号方式：

```json
{
  "start_station_id": 1,
  "end_station_id": 5,
  "preference": "balanced",
  "allow_transfer": true,
  "max_transfer_count": 2,
  "max_walk_minutes": 20
}
```

坐标方式：

```json
{
  "origin_longitude": 116.397,
  "origin_latitude": 39.908,
  "destination_longitude": 116.407,
  "destination_latitude": 39.918,
  "preference": "fastest"
}
```

`preference` 可选值：

```text
balanced / low_load / less_walking / less_transfer / fastest
```

### 4.6 AI 出行助手模块

| 接口名称 | 方法 | 路径 | 前端方法 | 鉴权 |
|---|---|---|---|---|
| AI 出行问答、建议与解释 | `POST` | `/api/v1/ai/travel` | `askAiTravel(data)` / `sendAiMessage(data)` | 否 |

问答模式：

```json
{
  "mode": "qa",
  "question": "现在去教学楼哪条线路不太挤？",
  "preference": "balanced"
}
```

建议模式：

```json
{
  "mode": "suggest",
  "start_station_id": 1,
  "end_station_id": 3,
  "preference": "balanced"
}
```

解释模式：

```json
{
  "mode": "explain",
  "route_id": "route_001"
}
```

`mode` 可选值：

```text
qa / suggest / explain
```

### 4.7 仿真与预测更新模块

| 接口名称 | 方法 | 路径 | 前端方法 | 鉴权 |

{

  "bus_stop_code": "01012",

## 5. 前端调用规范

页面组件中只调用 `frontend/src/api` 中的函数，不直接拼接完整后端地址。

推荐写法：

```js
import { recommendRoutes } from '@/api'

const response = await recommendRoutes({
  start_station_id: 1,
  end_station_id: 5,
  preference: 'balanced'
})

const routes = response.data?.items || []
```

不推荐写法：

```js
axios.post('http://127.0.0.1:8000/api/v1/recommend-routes', data)
```
