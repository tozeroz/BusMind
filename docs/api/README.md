
## 推荐路线完整链路

```text
前端提交推荐请求
    -> 后端寻路算法生成候选路线
    -> 后端补齐 ETA、Load、客流、路况、步行、换乘、频率等字段
    -> 后端调用推荐模型
    -> 模型预处理原始字段并映射为内部特征
    -> 模型返回分项评分和综合推荐分
    -> 后端按分组排序
    -> 前端展示推荐路线和解释
```

边界约定：

```text
后端负责候选路线生成和业务字段聚合。
模型只负责候选路线评分。
大模型助手只负责读取结构化推荐结果并生成自然语言解释。
```

## 本地联调入口

按根目录 `README.md` 启动 3 个 PowerShell 窗口：

```text
数据库 SSH 隧道 3307
FastAPI 后端 8001
Vue 前端 5173
```

如果线路或推荐接口返回 500，优先检查数据库隧道：

```powershell
Test-NetConnection 127.0.0.1 -Port 3307
```

看到 `TcpTestSucceeded : True` 才表示隧道正常。
