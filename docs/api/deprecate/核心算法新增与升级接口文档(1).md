# BusMind 推荐模型与核心算法接口文档

> 当前版本只覆盖推荐评分模型，不做 ETA / Load 训练。ETA 来自 LTA Bus Arrival 的 `EstimatedArrival - query_time`，Load 直接来自 LTA Bus Arrival 的 `Load` 字段。

## 1. 模型定位

当前模型是公交候选路线评分器：

```text
后端传入候选路线 + 12 个数值特征
    ↓
模型输出五维体验分 + 综合推荐分
    ↓
后端根据分数排序、打标签、组织公开响应
```

推荐调用入口：`algorithm.model.predictor.predict_recommendation(payload)`。

模型不负责：

- 生成候选路线；
- 调 LTA API；
- 查数据库；
- 预测 ETA；
- 预测 Load；
- 生成最终自然语言解释。

## 2. 模型输入

每条路线固定 12 个数值特征：

| 序号 | 字段 | 含义 |
|---:|---|---|
| 1 | `eta_minutes` | 实时 ETA |
| 2 | `ride_time_minutes` | 车内时间 |
| 3 | `walk_time_minutes` | 步行时间 |
| 4 | `walk_distance_meters` | 步行距离 |
| 5 | `transfer_count` | 换乘次数 |
| 6 | `load_score` | 客载舒适分 |
| 7 | `history_flow_score` | 历史站点客流分 |
| 8 | `congestion_score` | 道路通畅分 |
| 9 | `data_freshness_seconds` | 实时数据年龄 |
| 10 | `monitored_score` | LTA 监控可信度 |
| 11 | `completeness_score` | 特征完整度 |
| 12 | `avg_service_frequency_minutes` | 平均发车间隔 |

辅助字段：

```text
route_id
service_nos
load_code
feature_sources
degraded_fields
```

## 3. 模型输出

模型输出每条路线的五维分数：

```text
time_score
comfort_score
walk_score
transfer_score
reliability_score
recommend_score
```

后端根据这些分数生成：

```text
best_experience_route_id
fastest_route_id
least_crowded_route_id
least_walking_route_id
least_transfer_route_id
```

## 4. 模型结构

当前实现：

```text
x ∈ R^12
z = (x - mean) / std
s = sigmoid(C_sub · z + b_sub) × 100
recommend_score = preference_mix(preference) · s
```

其中：

- `x` 是 12 维路线特征；
- `z` 是标准化后的特征；
- `C_sub` 是 `5 x 12` 的评分头系数矩阵；
- `b_sub` 是 5 维偏置；
- `s` 是五维体验分；
- `preference_mix` 是模型内置偏好模板系数。

模型不接受后端传入 `weights`。后端只传 `preference`。

## 5. 偏好枚举

```text
balanced
fastest
comfort
less_walking
less_transfer
```

兼容：

```text
low_load -> comfort
```

## 6. 训练链路

```text
algorithm/dataset/scripts/build_features.py
    ↓
algorithm/dataset/features.csv
    ↓
algorithm/dataset/scripts/build_rule_pseudo_labels.py
    ↓
algorithm/dataset/rule_pseudo_labels.csv
    ↓
algorithm/model/linear_scoring/train.py
    ↓
algorithm/model/linear_scoring/artifacts/linear_route_scoring.json
```

训练第一版在缺少真实用户选择数据的情况下，使用独立规则生成伪标签作为冷启动弱监督，再拟合 12 维输入到 5 维评分头。

当前训练脚本输出：

```text
algorithm/model/linear_scoring/artifacts/linear_route_scoring.json
```

模型产物包含：

```text
feature_names
score_names
mean
std
subscore_coefficients
subscore_bias
training_rows
loss
```

## 7. 技术口径

对外说明建议：

> BusMind 推荐模块采用多目标路线评分模型。模型以 ETA、客载、步行、换乘、发车频率、历史客流、道路拥堵和数据可靠性等 12 个特征为输入，经过标准化和五维评分头得到时间效率、舒适度、步行体验、换乘体验、可靠性五类分数，再根据用户偏好生成综合推荐分。第一版在缺少真实用户选择数据的情况下，使用独立规则生成伪标签作为冷启动弱监督；后续可用真实用户反馈替换或校准伪标签，继续优化排序。
