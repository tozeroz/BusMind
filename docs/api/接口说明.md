# BusMind 前后端接口接入说明

> 适用范围：前端页面联调与接口替换  
> 后端 API 版本：v1.2.0  
> 更新时间：2026-07-11  
> 字段全集以 `code/docs/api/BusMind项目接口文档.md` 和运行时 `/docs` 为准

## 一、统一说明

### 1. 请求入口

前端统一通过 Axios 实例发送请求：

```text
frontend/src/api/request.js
```

默认基础路径为：

```js
import.meta.env.VITE_API_BASE_URL || '/api/v1'
```

开发环境可在 `.env` 中配置：

```text
VITE_API_BASE_URL=http://127.0.0.1:8000/api/v1
```

业务接口封装位于：

```text
frontend/src/api/
```

页面组件不应直接创建新的 Axios 实例，应调用该目录中已经封装的函数。

### 2. 鉴权

登录成功后，前端通过以下请求头携带身份：

```http
Authorization: Bearer <access_token>
```

规范 Token 存储键为：

```text
busmind_access_token
```

`access_token` 仅作为旧代码兼容键。用户资料、收藏、查询历史等个人接口需要登录。当前部分管理写接口仍处于演示联调状态，正式部署前需要增加管理员权限校验。

### 3. 统一响应与 Axios 解包

后端统一响应结构：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "trace_id": "req_xxx",
  "timestamp": "2026-07-11T20:00:00+08:00"
}
```

Axios 响应拦截器执行：

```js
(response) => response.data
```

因此页面拿到的是上述统一外壳，真正的业务数据仍在 `response.data`：

```js
const response = await getLines()

if (response.code !== 0) {
  throw new Error(response.message)
}

const businessData = response.data
```

集合字段并不完全统一，常见读取方式包括：

```text
response.data.lines
response.data.stations
response.data.vehicles
response.data.items
response.data.segments
```

请按照具体接口的数据模型读取，不要假设所有列表都直接位于 `response.data`。

### 4. 时间、分页和枚举

- 时间统一使用 ISO 8601，建议包含时区；
- 经纬度使用 WGS84；
- 分页参数以具体接口为准，常见为 `page`、`limit`；
- 推荐偏好使用 `balanced`、`comfort`（优先）、`low_load`（兼容别名）、`less_walking`、`less_transfer`、`fastest`；
- 新核心算法升级后 `low_load` 将兼容映射为 `comfort`，前端切换前需同步更新枚举；
- 客载状态常见为 `seats_available`、`standing_available`、`limited_standing`；
- AI 模式为 `qa`、`suggest`、`explain`。

### 5. 错误处理

| HTTP 状态 | 前端处理建议 |
|---:|---|
| 400 | 显示后端 `message`，提示业务参数错误 |
| 401 | 清理 Token 并跳转登录页 |
| 403 | 提示无权限，不自动重试 |
| 404 | 显示资源或推荐方案不存在 |
| 409 | 提示数据重复或编码冲突 |
| 422 | 检查请求字段、类型和必填参数 |
| 500 | 显示系统错误并记录 `trace_id` |
| 503 | 提示实时数据或 LTA 服务暂不可用 |

请求失败时，`request.js` 目前只输出警告并抛出异常，并不会自动返回 Mock 数据。页面若需 Mock 降级，必须在页面或数据适配层显式处理。

## 三、核心页面接口

### 1. 用户登录

```http
POST /api/v1/users/login
Content-Type: application/json
```

请求：

```json
{
  "username": "demo_user",
  "password": "12345678"
}
```

前端调用：

```js
import { loginUser } from '@/api/user'

const response = await loginUser(form)
const result = response.data
```

登录成功后，将响应中的访问令牌保存到 `busmind_access_token`。令牌具体字段以运行时 `/docs` 的登录响应 Schema 为准。

### 2. 位置搜索

```http
GET /api/v1/locations/search?keyword=airport
```

前端调用：

```js
import { searchLocations } from '@/api/location'

const response = await searchLocations({ keyword: searchText })
const result = response.data
```

页面应对空列表正常展示“未找到地点”，不要直接切换成固定 Mock 结果。

### 3. 地图站点、线路和道路

```http
GET /api/v1/map/stations
GET /api/v1/map/lines
GET /api/v1/map/road-segments
```

前端封装：

```js
getMapStations(params)
getMapLines()
getRoadSegments()
```

主要用于 `BusMap.vue`。页面应先判断 `response.code === 0`，再读取 `response.data` 中对应集合。

### 4. 路线推荐

```http
POST /api/v1/recommend-routes
Content-Type: application/json
```

站点模式请求：

```json
{
  "start_station_id": 1,
  "end_station_id": 8,
  "preference": "balanced",
  "allow_transfer": true,
  "max_transfer_count": 1
}
```

坐标模式请求：

```json
{
  "origin_longitude": 103.8519,
  "origin_latitude": 1.2903,
  "destination_longitude": 103.8198,
  "destination_latitude": 1.3521,
  "preference": "less_walking",
  "allow_transfer": true,
  "max_transfer_count": 1
}
```

不能只提供半组坐标。前端调用：

```js
import { recommendRoutes } from '@/api/intelligence'

const response = await recommendRoutes(payload)
const result = response.data
const routes = result.items
```

当前推荐结果关键字段：

```text
items
best_experience_route_id
fastest_route_id
least_crowded_route_id
least_walking_route_id
least_transfer_route_id
preference
generated_at
```

每条 `items[]` 主要包含：

```text
route_id
line_ids
segments
boarding_station
alighting_station
predicted_eta_minutes
predicted_load
walk_time_minutes
ride_time_minutes
total_time_minutes
transfer_count
experience_score
recommend_types
reason
```

后续五维评分升级字段见：

```text
code/docs/api/核心算法新增与升级接口文档.md
```

### 5. 实时 ETA

```http
GET /api/v1/eta?vehicle_id=101&target_station_id=1&line_id=1
```

前端调用：

```js
import { getEta } from '@/api/intelligence'

const response = await getEta({
  vehicle_id: 101,
  target_station_id: 1,
  line_id: 1
})

const eta = response.data
```

当前响应字段仍使用 `predicted_eta_minutes`。后续切换为 `eta_minutes` 时需要同步修改前端映射。

### 6. 车辆客载

```http
POST /api/v1/passenger-load-prediction
Content-Type: application/json
```

```json
{
  "line_id": 1,
  "station_id": 1,
  "vehicle_id": 101
}
```

注意：Passenger Load 表示车辆内客载状态，不等于 Passenger Flow 站点或线路客流。

### 7. AI 出行助手

```http
POST /api/v1/ai/travel
Content-Type: application/json
```

推荐模式示例：

```json
{
  "mode": "suggest",
  "start_station_id": 1,
  "end_station_id": 8,
  "preference": "low_load"
}
```

响应关键字段：

```text
answer
mode
used_tools
related_routes
reminders
fallback
```

`fallback=true` 表示未使用外部大模型，但可能已经返回本地结构化推荐结果，前端不应把它当成接口失败。

## 四、线路、站点与车辆接口

### 1. 线路

```http
GET    /api/v1/lines
GET    /api/v1/lines/{line_id}
POST   /api/v1/lines
PATCH  /api/v1/lines/{line_id}
DELETE /api/v1/lines/{line_id}

GET    /api/v1/lines/{line_id}/stations
POST   /api/v1/lines/{line_id}/stations
PATCH  /api/v1/lines/stations/{line_station_id}
DELETE /api/v1/lines/stations/{line_station_id}
```

线路列表页读取后端返回的线路集合；详情页应以 `line_id` 请求线路基础信息和站序，不要使用列表页缓存代替详情响应。

### 2. 站点

```http
GET    /api/v1/stations
GET    /api/v1/stations/{station_id}
POST   /api/v1/stations
PATCH  /api/v1/stations/{station_id}
DELETE /api/v1/stations/{station_id}
GET    /api/v1/stations/{station_id}/lines
POST   /api/v1/stations/nearby
GET    /api/v1/stations/coordinates/all
```

附近站点请求：

```json
{
  "latitude": 1.2903,
  "longitude": 103.8519
}
```

### 3. 车辆

```http
GET    /api/v1/vehicles
GET    /api/v1/vehicles/realtime
GET    /api/v1/vehicles/line/{line_id}
GET    /api/v1/vehicles/{vehicle_id}
POST   /api/v1/vehicles
PATCH  /api/v1/vehicles/{vehicle_id}
DELETE /api/v1/vehicles/{vehicle_id}
```

车辆数据库接口速度字段可能使用 `speed_kmh`，智能/仿真接口使用 `speed_kph`，前端适配层需要显式映射。

## 五、个人中心接口

```http
GET    /api/v1/users/me
PATCH  /api/v1/users/me
GET    /api/v1/users/me/favorites
POST   /api/v1/users/me/favorites
DELETE /api/v1/users/me/favorites/{favorite_id}
GET    /api/v1/users/me/query-history
```

以上接口均应携带 Bearer Token。收藏接口的目标类型、目标编号等字段以 `BusMind项目接口文档.md` 为准，不应由页面自由拼接枚举。

## 六、历史数据接口

```http
GET /api/v1/history/passenger-flow          ← 第一阶段核心：历史客流趋势
GET /api/v1/history/passenger-flow/prediction  ← 兼容/实验接口，非第一阶段核心能力
GET /api/v1/history/eta/line/{line_id}
GET /api/v1/history/eta/{vehicle_id}/{target_station_id}
GET /api/v1/history/load/line/{line_id}
GET /api/v1/history/load/{line_id}
```

历史 Passenger Flow 是站点/线路客流；历史 Load 是车辆客载。页面名称、图表图例和字段命名不要混用。

## 七、管理与仿真接口

### 1. 正式 LTA 管理刷新入口

```http
POST /api/v1/admin/lta/bus-arrival/refresh
POST /api/v1/admin/lta/traffic-speed-bands/refresh
```

Bus Arrival 请求示例：

```json
{
  "bus_stop_code": "83139",
  "service_no": "15",
  "sync_to_db": true
}
```

Traffic Speed Bands 请求示例：

```json
{
  "sync_to_db": true
}
```

调用这些接口需要后端配置 `LTA_ACCOUNT_KEY`。密钥只能保存在后端环境变量中，前端不得携带或保存。

### 2. 仿真接口

```http
PATCH /api/v1/simulation/vehicle-status/{vehicle_id}
POST  /api/v1/simulation/prediction-results
```

旧接口：

```http
POST /api/v1/simulation/lta-bus-arrival/refresh
```

已经废弃，前端不得继续新增调用。现有 `frontend/src/api/simulation.js` 中的旧封装应迁移到 `frontend/src/api/admin.js` 的正式管理接口。

## 九、前端替换 Mock 的标准流程

页面替换 Mock 时按以下顺序执行：

1. 在 `frontend/src/api/` 中确认或新增接口封装；
2. 使用后端实际请求字段构造参数；
3. 判断 `response.code`；
4. 从 `response.data` 读取业务对象；
5. 在适配层统一字段名，不在多个组件中重复转换；
6. 分别处理加载、空数据、业务失败和网络失败；
7. 联调通过后删除该页面对应的固定 Mock 分支；
8. 不要在 `catch` 中无提示地永久展示 Mock 数据。

推荐写法：

```js
async function loadRoutes(payload) {
  loading.value = true
  errorMessage.value = ''

  try {
    const response = await recommendRoutes(payload)
    if (response.code !== 0) {
      throw new Error(response.message || '路线推荐失败')
    }
    routes.value = response.data.items || []
  } catch (error) {
    routes.value = []
    errorMessage.value = error?.message || '网络连接失败'
  } finally {
    loading.value = false
  }
}
```

## 十、联调前检查清单

- [ ] 后端已启动，`GET /api/v1/` 返回成功；
- [ ] 浏览器可以访问 `/docs`；
- [ ] `VITE_API_BASE_URL` 指向 `/api/v1`；
- [ ] 前端没有重复拼接 `/api/v1`；
- [ ] 需要登录的接口带 Bearer Token；
- [ ] 页面从统一响应的 `data` 字段读取业务数据；
- [ ] 列表集合键与具体接口一致；
- [ ] 请求枚举与后端 Schema 一致；
- [ ] 推荐请求提供完整站点 ID 对或完整坐标对；
- [ ] 管理刷新使用 `/admin/lta/*`，不再使用旧 simulation 刷新接口；
- [ ] 空数据、401、422、503 均有明确展示；
- [ ] Mock 替换状态已经在页面清单中更新。
