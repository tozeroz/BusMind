# BusMind 模型数据目录职责说明

> 当前口径：模型侧数据路径分为原始采集层、业务标准化层、模型训练数据层。三层职责不能混用。

## 1. 总流程

```text
LTA API
    ↓
data/raw
    ↓
data/processed
    ↓
algorithm/dataset
    ↓
algorithm/model
```

## 2. 三个目录的固定职责

| 目录 | 定位 | 输入 | 输出 | 不负责 |
|---|---|---|---|---|
| `data/raw` | LTA 原始数据层 | LTA DataMall API | 原始 JSON、JSONL、ZIP | 清洗字段、生成模型特征、训练标签 |
| `data/processed` | 后端业务标准化层 | `data/raw/lta` | 数据库导入和业务查询用 CSV | 模型伪标签、模型训练产物、用户偏好排序 |
| `algorithm/dataset` | 推荐模型训练数据层 | 优先 `data/processed`，缺失时回退 `data/raw/lta` | `features.csv`、`pseudo_labels.csv` | LTA 采集、MySQL 写入、线上缓存 |

## 3. data/raw

`data/raw` 只保存原始数据和采集脚本。

允许内容：

- LTA 静态公交数据：Bus Stops、Bus Routes、Bus Services。
- Passenger Volume 原始 ZIP。
- Bus Arrival 采样 JSONL。
- Traffic Speed Bands 原始 JSON。
- 采集脚本和原始数据检查脚本。

禁止职责：

- 不清洗字段。
- 不改业务 ID。
- 不生成推荐模型 `features.csv`。
- 不保存训练标签或模型产物。

## 4. data/processed

`data/processed` 是从 raw 到后端数据库的标准化中间层。

允许职责：

- 字段重命名和类型转换。
- 生成 `station_id`、`line_id`、`line_station_id` 等可导入数据库的 ID。
- 扁平化 Bus Arrival 的 ETA、Load、车辆状态。
- 聚合 Passenger Volume 历史客流。
- 整理 Traffic Speed Bands 道路热力。

典型输出：

```text
bus_station.csv
bus_line.csv
line_station.csv
lta_bus_arrival.csv
bus_eta_status.csv
bus_load_status.csv
passenger_flow_trend.csv
traffic_speed_bands.csv
```

注意：这些 CSV 是后端业务数据表口径，不是最终模型训练样本。ETA 和 Load 仍然来自 LTA 实时字段，不是 ETA/Load 预测模型标签。

## 5. algorithm/dataset

`algorithm/dataset` 是推荐模型训练数据层。

允许职责：

- 从 `data/processed` 读取业务 CSV。
- processed 缺失时，从 `data/raw/lta` 按相同口径重建。
- 构造推荐模型 12 维输入特征。
- 输出 `algorithm/dataset/recommendation/v1/features.csv`。
- 基于规则基线生成 `pseudo_labels.csv`。
- 标记 `is_synthetic`、`feature_sources`、`degraded_fields`。

禁止职责：

- 不调用 LTA API。
- 不写 MySQL。
- 不管理线上缓存。
- 不保存模型参数文件。

当前推荐模型 v1 的训练链路：

```text
algorithm/dataset/build_features.py
    ↓
algorithm/dataset/recommendation/v1/features.csv
    ↓
algorithm/dataset/build_labels.py
    ↓
algorithm/dataset/recommendation/v1/pseudo_labels.csv
    ↓
algorithm/model/train_ranker.py
    ↓
algorithm/model/artifacts/route_scorer_v1.json
```

## 6. 12 维特征来源归类

真实来源优先：

- `eta_minutes`
- `load_score`
- `history_flow_score`
- `congestion_score`
- `avg_service_frequency_minutes`

当前离线阶段可估算：

- `ride_time_minutes`
- `walk_time_minutes`
- `walk_distance_meters`
- `transfer_count`
- `data_freshness_seconds`
- `completeness_score`

估算字段必须写入 `feature_sources` 和 `degraded_fields`，避免训练样本和真实线上特征混淆。

## 7. 提交规则

- 采集脚本、处理脚本、模型数据构建脚本可以提交。
- 真实 raw/processed 数据文件通常不提交。
- `algorithm/dataset/recommendation/v1` 下的大规模训练样本通常不提交。
- 如果为了演示保留小样本，应明确命名为 sample，并在文档中说明不是正式训练数据。
