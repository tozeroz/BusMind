# 评分模型选型与对比

## 1. 问题背景

当前项目需要解决的是**候选线路评分模型**问题：后端先生成若干候选线路，并为每条候选线路聚合 ETA、乘车时间、步行、换乘、客载、历史客流、道路拥堵、数据新鲜度等业务字段；模型侧再将这些字段转换为固定的 12 维数值特征，并输出多维体验评分

目标输出包括：

```text
time_score
comfort_score
walk_score
transfer_score
reliability_score
recommend_score
```

其中前五个分数描述候选线路在不同体验维度上的表现，`recommend_score` 用于后端最终排序和推荐展示

当前难点是缺少真实用户选择、点击、满意度等监督标签，因此第一阶段需要依赖规则伪标签或 LLM-assisted pseudo-labeling 构造训练目标。模型选型的核心不是单纯追求复杂度，而是比较不同模型在小数据、伪标签、结构化特征和可部署性条件下的效果

本文比较三类候选方案：

- 当前 12 维特征线性 Sigmoid 多目标评分模型
- XGBoost 结构化特征回归模型
- TabPFN 小数据集表格学习模型

## 2. 12 维特征线性 Sigmoid 多目标评分模型

当前线上可用模型是项目已有的候选线路评分模型，结构可以概括为：

```text
route payload
    ↓
preprocessing
    ↓
x ∈ R^12
    ↓
z = (x - mean) / std
    ↓
scores = sigmoid(Wz + b) * 100
    ↓
five_scores + recommend_score
```

其中 12 个输入特征为：

```text
eta_minutes
ride_time_minutes
walk_time_minutes
walk_distance_meters
transfer_count
load_score
history_flow_score
congestion_score
data_freshness_seconds
monitored_score
completeness_score
avg_service_frequency_minutes
```

模型本体是 `5 x 12` 的线性评分头，分别预测：

```text
time_score
comfort_score
walk_score
transfer_score
reliability_score
```

`recommend_score` 不是单独学习出来的，而是根据用户偏好权重对五维分数加权得到

优点：

- 结构简单，推理速度快
- 参数可解释，容易定位每个特征对评分的影响
- 适合作为第一版可运行 baseline
- 对数据量要求低，适合冷启动阶段
- 线上部署成本低，不依赖复杂模型运行环境

局限：

- 表达能力有限，只能学习近似线性关系
- 很难捕捉特征之间的复杂交互，例如“高 ETA + 高拥挤 + 多换乘”的组合惩罚
- 对伪标签规则依赖较强，容易学到规则表面
- 当训练数据质量提升后，模型上限可能不足

因此，当前模型适合作为**可解释基线模型**，但不一定是最终效果最好的方案

## 3. XGBoost：面向结构化特征的强基线模型

XGBoost 是梯度提升树模型，适合处理结构化表格特征。对于当前候选线路评分任务，可以将 12 维特征作为输入，分别训练 5 个回归器：

```text
X ∈ R^12
    ↓
XGBoostRegressor(time_score)
XGBoostRegressor(comfort_score)
XGBoostRegressor(walk_score)
XGBoostRegressor(transfer_score)
XGBoostRegressor(reliability_score)
```

随后仍然由系统根据 preference 权重计算：

```text
recommend_score = preference_weights · five_scores
```

也可以使用多输出包装器统一训练，但从可控性和调试角度看，第一版更建议五个目标分别训练

优点：

- 对表格数据非常强，是结构化特征任务的经典强基线
- 能自动学习非线性关系和特征交互
- 对不同量纲、异常值和部分噪声通常较稳健
- 训练和推理成本可控，工程部署成熟
- 可通过 feature importance、SHAP 等方式解释模型

局限：

- 相比当前线性模型，可解释性下降
- 如果伪标签质量较差，XGBoost 可能更强地拟合伪标签噪声
- 需要调参，例如树深、学习率、子采样比例、正则项等
- 多目标评分需要维护多个回归器或多输出封装

因此，XGBoost 适合作为**主要增强模型**，尤其适合在伪标签质量提高后替代当前线性评分头

## 4. TabPFN：面向小数据集表格学习的候选方案

TabPFN 是面向小数据集表格任务的模型思路。它的核心价值是：在样本量不大、特征数量有限的表格任务上，利用预训练得到的表格先验快速完成预测，而不是像传统模型那样完全从当前数据集开始学习

对于当前任务，可以把 TabPFN 作为小数据集条件下的实验候选：

```text
12 维候选线路特征
    ↓
TabPFNRegressor
    ↓
五维评分目标之一
```

第一版同样建议为五个评分目标分别训练或调用五个 TabPFN 回归任务，而不是一开始就强行做复杂多目标封装

优点：

- 适合小规模表格数据实验
- 对少样本场景可能比传统模型更有优势
- 训练流程相对轻，适合做快速对比
- 可以作为 XGBoost 之外的现代表格模型参照

局限：

- 对样本量、特征数、环境和版本可能有更强约束
- 工程部署成熟度通常不如 XGBoost
- 模型解释性较弱
- 在伪标签噪声较大时，不一定稳定优于 XGBoost
- 若数据量继续增长，TabPFN 的适用性需要重新评估

因此，TabPFN 更适合作为**小数据集实验模型**，用于验证现代表格学习方法是否能在当前 12 维特征任务上带来收益

## 5. 三类模型的对比

| **维度** | **当前线性 Sigmoid 模型** | **XGBoost** | **TabPFN** |
|---|---|---|---|
| 模型定位 | 可解释 baseline | 结构化特征强基线 | 小数据集表格模型候选 |
| 输入 | 12 维固定特征 | 12 维固定特征 | 12 维固定特征 |
| 输出 | 五维评分 + 加权推荐分 | 五维评分 + 加权推荐分 | 五维评分 + 加权推荐分 |
| 非线性能力 | 弱 | 强 | 强 |
| 特征交互能力 | 弱 | 强 | 较强 |
| 小数据适应性 | 较好 | 较好 | 强 |
| 可解释性 | 强 | 中等 | 弱 |
| 部署复杂度 | 低 | 中 | 中到高 |
| 调参成本 | 低 | 中 | 中 |
| 抗伪标签噪声能力 | 中等 | 取决于正则和数据质量 | 待实验验证 |
| 推荐用途 | 保底模型、解释模型 | 主力候选模型 | 对比实验模型 |

从工程角度看，三者不是互斥关系，而是递进关系：

```text
当前线性模型
    ↓
作为可解释 baseline

XGBoost
    ↓
作为主要效果增强方案

TabPFN
    ↓
作为小数据集现代方法对照
```

## 6. 实验设计

实验目标不是只比较训练集误差，而是判断哪类模型更适合作为候选线路评分模型

### 6.1 数据准备

输入数据：

```text
algorithm/dataset/features.csv
algorithm/dataset/pseudo_labels.csv
```

处理流程：

```text
features.csv
    ↓
algorithm/dataset/scripts/recommendation_feature_contract.py
    ↓
12 维 numeric features
    ↓
与 pseudo_labels.csv 按 candidate_group_id + route_id 对齐
```

注意事项：

- 五维评分目标本身不应按 preference 重复训练 5 倍
- 如果 `pseudo_labels.csv` 对同一候选线路包含多个 preference，应先按 `candidate_group_id + route_id` 去重，提取稳定的五维 subscore
- `recommend_score` 可以由五维评分和 preference 权重计算，不一定作为独立训练目标
- 划分训练集和测试集时应按 `candidate_group_id` 划分，避免同一组候选线路同时出现在训练集和测试集造成泄漏

### 6.2 模型设置

建议比较三组模型：

```text
Baseline A：当前 12 维线性 Sigmoid 候选线路评分模型
Baseline B：XGBoost 五目标回归模型
Baseline C：TabPFN 五目标回归模型
```

每个模型统一输出：

```text
time_score
comfort_score
walk_score
transfer_score
reliability_score
recommend_score
```

### 6.3 指标设计

评分回归指标：

| **指标** | **用途** |
|---|---|
| MAE | 平均绝对误差，直观看每个评分差多少分 |
| RMSE | 对大误差更敏感，观察极端预测偏差 |
| Pearson | 观察预测分数和伪标签分数的线性相关性 |
| Spearman | 观察预测排序趋势是否一致 |

推荐排序指标：

| **指标** | **用途** |
|---|---|
| Top-1 一致率 | 每组候选线路中，模型选出的第一名是否和伪标签一致 |
| NDCG@K | 评估候选组内整体排序质量 |
| Kendall / Spearman rank | 评估组内排序相关性 |

稳定性指标：

| **指标** | **用途** |
|---|---|
| 不同随机种子结果方差 | 判断模型训练稳定性 |
| 特征扰动前后分数变化 | 判断模型鲁棒性 |
| 缺失字段 / degraded_fields 场景表现 | 判断降级样本是否可控 |

## 7. 阶段性结论与选型建议

当前阶段建议采用“三层对比”思路：

1. 保留当前 12 维线性 Sigmoid 候选线路评分模型作为 baseline
2. 引入 XGBoost 作为主要效果增强模型
3. 引入 TabPFN 作为小数据集表格学习对照模型

阶段性判断：

- 如果 XGBoost 在 MAE、Top-1、NDCG 上明显优于当前模型，且推理成本可接受，可以优先升级为主力候选线路评分模型
- 如果 TabPFN 在小数据集上明显优于 XGBoost，可以保留为实验方向，但上线前仍需评估依赖、推理速度和稳定性
- 如果三者效果差距不大，应优先保留当前线性模型，因为它更简单、更稳定、更易解释
- 如果模型之间差异主要来自伪标签质量，则应优先优化 pseudo-labeling，而不是盲目换更复杂的模型

更合理的路线是：

```text
先提高伪标签质量
    ↓
再比较 Linear / XGBoost / TabPFN
    ↓
最后根据指标、解释性和部署成本选择主力候选线路评分模型
```
