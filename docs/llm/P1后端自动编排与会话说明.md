# P1 后端自动编排与会话说明

## 1. 完成范围

P1 只修改后端 AI 链路、测试和本目录文档，没有修改 `frontend/`，也没有修改任何 `algorithm/` 目录代码。

已完成：

- `mode` 可省略，由后端规则优先识别 `qa/suggest/explain`；
- 显式传入 `mode` 的 P0 调用保持兼容；
- 从自然语言提取起点站编号、终点站编号、偏好、路线编号和“第 N 条”；
- 参数不足时返回 `needs_clarification`、`missing_fields` 和已解析槽位；
- 每次响应返回 `conversation_id`；
- 保存最近一次推荐快照和当前选中路线；
- 支持“推荐路线 → 为什么 → 换一条”，后两步不要求前端重复传完整路线；
- “换一条”直接轮换快照，不重复调用推荐服务。

## 2. 请求与响应新增字段

请求示例：

```json
{
  "question": "从站点1到站点12怎么走，想舒适一点",
  "conversation_id": null
}
```

- `mode`：现在可选；省略时由后端识别，显式传值仍优先；
- `conversation_id`：首次请求不传，后续请求原样回传；
- 起终点自然语言解析当前只承诺识别站点编号，例如“从站点1到站点12”。站名到站点编号仍应使用已有站点选择能力，不在本阶段猜测站名。

响应新增字段示例：

```json
{
  "mode": "suggest",
  "conversation_id": "c9ea1cc3d22247b99a92d6892c57ced1",
  "resolved_slots": {
    "action": "recommend",
    "start_station_id": 1,
    "end_station_id": 12,
    "preference": "comfort",
    "route_id": null
  }
}
```

`resolved_slots` 用于界面展示或调试后端实际理解的参数，不要求前端据此再次判断意图。

## 3. 多轮调用示例

首次推荐：

```json
POST /api/v1/ai/travel
{
  "question": "从站点1到站点12怎么走"
}
```

解释上次默认选中的路线：

```json
POST /api/v1/ai/travel
{
  "question": "为什么推荐这条？",
  "conversation_id": "<首次响应中的 conversation_id>"
}
```

切换到下一条候选路线：

```json
POST /api/v1/ai/travel
{
  "question": "换一条",
  "conversation_id": "<同一个 conversation_id>"
}
```

三次响应的 `conversation_id` 相同。解释和换路线使用 `conversation_snapshot`，不要求传 `context.items`。

## 4. 结构化追问续接

用户只说“从站点1怎么走”时，返回：

```json
{
  "status": "needs_clarification",
  "missing_fields": ["end_station_id"],
  "conversation_id": "<新会话>",
  "resolved_slots": {
    "start_station_id": 1,
    "end_station_id": null
  }
}
```

前端将同一 `conversation_id` 随“到站点12”回传，后端会继承起点并执行推荐。

## 5. 前端联调最小改动

前端负责人只需：

1. 首次调用允许不传 `mode`；现有显式 `mode` 调用可以暂时保留；
2. 保存响应中的 `conversation_id`，同一聊天窗口后续请求原样回传；
3. `needs_clarification` 时展示 `answer`，无需在前端自行判断缺哪个参数；
4. 新建对话或清空聊天时丢弃旧 `conversation_id`；
5. 继续使用现有站点选择组件提供准确站点编号，避免依赖自由文本猜站名。

## 6. 当前存储边界

会话快照保存在后端进程内的 LRU 存储中，最多保留 500 个最近会话：

- 服务重启后快照消失；
- 多进程实例之间不共享；
- 不保存聊天全文，只保存最近路线、起终点、偏好和选中路线；
- 当前适用于单进程开发和前端联调，不作为长期持久化方案。

生产部署需要多进程共享或跨重启恢复时，将同一存储接口替换为 Redis，不需要改变前端请求格式。
