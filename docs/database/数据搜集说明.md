> 推荐方案：线路、站点、坐标尽量使用真实地图数据；车辆运行、历史客流、拥挤等级和预测结果用规则模拟生成

# 一、数据来源总策略

本项目不需要寻找完整的真实公交运营数据，真实公交的载客人数、GPS、调度和刷卡数据通常不公开，也不适合短期实训

建议采用：

```text
真实地图线路站点数据 + 模拟车辆运行数据 + 模拟历史客流数据 + 拥挤预测模型
```

这样既能保证地图和站点真实，又能保证系统演示稳定

# 三、地图 API 能做什么

可以调用地图相关 API 来做真实线路和站点数据，推荐优先使用高德地图开放平台

地图 API 适合获取：

```text
公交线路名称
线路 ID
起点站
终点站
途经站点
站点顺序
站点经纬度
首班时间
末班时间
线路轨迹 polyline
```

这些数据可以整理成：

```text
bus_line.csv
bus_station.csv
line_station.csv
```

地图 API 不适合获取：

```text
真实车上人数
真实拥挤程度
真实历史客流
真实车辆实时位置
真实调度数据
```

这些仍然需要你用规则模拟

# 四、推荐搜集方式

## (一) 优先方式：高德地图 API

操作流程：

```text
1. 注册高德开放平台账号
2. 创建 Web 服务 API Key
3. 选定城市或学校所在区域
4. 查询 3 条真实公交线路
5. 获取每条线路的站点列表和经纬度
6. 保存为 CSV
7. 人工补充站点类型、站间耗时、车辆容量等字段
```

注意：

- API Key 不要提交到 Git
- 建议用本地脚本一次性拉取并保存 CSV
- 系统演示时读取本地 CSV 或数据库，不要依赖每次实时请求地图 API
- 如果前端用高德地图，坐标可直接使用高德返回的坐标

## (二) 备用方式：地图手工整理

如果 API 来不及申请，可以直接手工整理：

```text
1. 打开高德地图或百度地图
2. 搜索学校、园区或附近公交线路
3. 记录线路名称和站点顺序
4. 用坐标拾取器记录站点经纬度
5. 手工估算站间距离和平均耗时
```

这种方式最快，适合第一天完成初版数据

# 八、拥挤等级规则

第一版直接用规则生成标签：

```text
crowd_rate = onboard_count / capacity

0.00 <= crowd_rate < 0.40：low
0.40 <= crowd_rate < 0.75：medium
0.75 <= crowd_rate <= 1.00：high
```

后续可以用 `crowd_history.csv` 训练随机森林模型，输入时间、线路、站点、天气、上车人数、下车人数、当前人数等特征，输出 `crowd_level`
