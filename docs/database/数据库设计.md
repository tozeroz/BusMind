# BusMind 项目最终数据库表设计文档

> 适用项目：BusMind 公交舒适出行系统  
> 技术栈：FastAPI + MySQL + Vue3  
> 目标：覆盖用户、线路、站点、车辆、地图、ETA、客载状态、客流趋势、客流预测、路线推荐、AI 助手、管理员统计及数据更新接口。

---

# 3. 必须建立的数据表

## 3.1 `user_account` 用户账号表

### 用途

用于注册、登录、身份判断、用户信息查询和管理员鉴权。

### 对应接口

```text
POST /api/v1/users/register
POST /api/v1/users/login
GET  /api/v1/users/me
POST /api/user/login
```

### 字段设计

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `user_id` | BIGINT | PK, AUTO_INCREMENT | 用户主键 |
| `username` | VARCHAR(50) | NOT NULL, UNIQUE | 登录用户名 |
| `password_hash` | VARCHAR(255) | NOT NULL | 密码哈希，禁止保存明文 |
| `nickname` | VARCHAR(50) | NULL | 用户昵称 |
| `role` | VARCHAR(20) | NOT NULL, DEFAULT `passenger` | `passenger`、`admin` |
| `status` | VARCHAR(20) | NOT NULL, DEFAULT `active` | `active`、`disabled` |
| `created_at` | DATETIME | NOT NULL | 注册时间 |
| `updated_at` | DATETIME | NOT NULL | 更新时间 |
| `last_login_at` | DATETIME | NULL | 最后登录时间 |

### 索引

```sql
UNIQUE KEY uk_user_username (username);
KEY idx_user_role (role);
KEY idx_user_status (status);
```

## 3.2 `user_query_history` 用户查询历史表

### 用途

记录路线查询、地点搜索、站点查询和 AI 问答。

### 字段设计

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `history_id` | BIGINT | PK, AUTO_INCREMENT | 主键 |
| `user_id` | BIGINT | FK, NULL | 用户 ID，游客可为空 |
| `query_type` | VARCHAR(30) | NOT NULL | 查询类型 |
| `origin_name` | VARCHAR(100) | NULL | 起点名称 |
| `origin_longitude` | DECIMAL(10,7) | NULL | 起点经度 |
| `origin_latitude` | DECIMAL(10,7) | NULL | 起点纬度 |
| `destination_name` | VARCHAR(100) | NULL | 终点名称 |
| `destination_longitude` | DECIMAL(10,7) | NULL | 终点经度 |
| `destination_latitude` | DECIMAL(10,7) | NULL | 终点纬度 |
| `selected_route_id` | BIGINT | NULL | 最终选择的线路或方案 ID |
| `query_content` | TEXT | NULL | 搜索词或 AI 问题 |
| `result_summary` | TEXT | NULL | 查询结果摘要 |
| `created_at` | DATETIME | NOT NULL | 查询时间 |

### 索引

```sql
KEY idx_history_user_time (user_id, created_at);
KEY idx_history_type (query_type);
```

## 3.5 `bus_line` 公交线路表

### 用途

线路列表、线路详情、运营信息及管理员线路统计。

### 字段设计

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `line_id` | BIGINT | PK | 线路主键，使用数据脚本生成的整数 ID |
| `service_no` | VARCHAR(30) | NOT NULL | 原始线路编号 |
| `line_name` | VARCHAR(100) | NOT NULL | 展示名称 |
| `operator` | VARCHAR(100) | NULL | 运营方 |
| `direction` | TINYINT | NOT NULL, DEFAULT 1 | 线路方向 |
| `origin_station_id` | BIGINT | FK, NULL | 起点站 |
| `destination_station_id` | BIGINT | FK, NULL | 终点站 |
| `am_peak_freq` | VARCHAR(20) | NULL | 早高峰发车间隔范围，保留 LTA 原始值，例如 `06-08` |
| `am_offpeak_freq` | VARCHAR(20) | NULL | 日间非高峰发车间隔范围，保留 LTA 原始值 |
| `pm_peak_freq` | VARCHAR(20) | NULL | 晚高峰发车间隔范围，保留 LTA 原始值 |
| `pm_offpeak_freq` | VARCHAR(20) | NULL | 晚间非高峰发车间隔范围，保留 LTA 原始值 |
| `avg_service_frequency` | DECIMAL(6,2) | NULL | 综合平均发车间隔，分钟，用于排序、推荐和展示 |
| `status` | VARCHAR(20) | NOT NULL, DEFAULT `running` | `running`、`suspended`、`offline` |
| `created_at` | DATETIME | NOT NULL | 创建时间 |
| `updated_at` | DATETIME | NOT NULL | 更新时间 |

### 索引

```sql
KEY idx_line_service_no (service_no);
KEY idx_line_name (line_name);
KEY idx_line_status (status);
```

| `updated_at` | DATETIME | NOT NULL | 更新时间 |

### 约束与索引

```sql

```

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|

| `line_id` | BIGINT | FK, NOT NULL | 所属线路 |
| `current_station_id` | BIGINT | FK, NULL | 当前站点 |
| `next_station_id` | BIGINT | FK, NULL | 下一站 |
| `next_station_name` | VARCHAR(100) | NULL | 下一站名称快照 |
| `current_position_text` | VARCHAR(150) | NULL | 如“宿舍区 → 食堂” |
| `longitude` | DECIMAL(10,7) | NULL | 经度 |
| `latitude` | DECIMAL(10,7) | NULL | 纬度 |
| `speed_kph` | DECIMAL(6,2) | NULL | 当前或估算速度 |
| `onboard_count` | INT | NULL | 车内人数 |
| `capacity` | INT | NULL | 车辆容量 |
| `predicted_load_level` | VARCHAR(30) | NULL | 客载状态 |
| `operation_status` | VARCHAR(20) | NOT NULL, DEFAULT `normal` | `normal`、`delayed`、`offline` |
| `delay_minutes` | INT | NOT NULL, DEFAULT 0 | 延误分钟数 |
| `data_status` | VARCHAR(20) | NOT NULL, DEFAULT `estimated` | `realtime`、`estimated` |
| `last_reported_at` | DATETIME | NULL | 最后上报时间 |
| `created_at` | DATETIME | NOT NULL | 创建时间 |
| `updated_at` | DATETIME | NOT NULL | 更新时间 |

### 索引

```sql
UNIQUE KEY uk_vehicle_code (vehicle_code);
KEY idx_vehicle_line (line_id);
KEY idx_vehicle_next_station (next_station_id);
KEY idx_vehicle_status (operation_status);
KEY idx_vehicle_updated (updated_at);
```

## 3.8 `eta_prediction` ETA 预测结果表

### 用途

存储车辆到指定站点的预计到站时间。

### 字段设计

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `eta_prediction_id` | BIGINT | PK, AUTO_INCREMENT | 主键 |
| `vehicle_id` | BIGINT | FK, NOT NULL | 车辆 ID |
| `line_id` | BIGINT | FK, NOT NULL | 线路 ID |
| `target_station_id` | BIGINT | FK, NOT NULL | 目标站点 |
| `prediction_time` | DATETIME | NOT NULL | 预测生成时间 |
| `predicted_eta_minutes` | DECIMAL(8,2) | NOT NULL | 预计到站分钟数 |
| `arrival_time` | DATETIME | NULL | 预计到站时间 |
| `vehicle_to_stop_distance_m` | DECIMAL(10,2) | NULL | 车辆到站距离 |
| `speed_kph` | DECIMAL(6,2) | NULL | 预测使用速度 |
| `confidence` | DECIMAL(5,4) | NULL | 置信度 |
| `model_version` | VARCHAR(100) | NULL | 模型版本 |
| `created_at` | DATETIME | NOT NULL | 入库时间 |
| `updated_at` | DATETIME | NOT NULL | 更新时间 |

### 索引

```sql
KEY idx_eta_vehicle_station_time (vehicle_id, target_station_id, prediction_time);
KEY idx_eta_line_station (line_id, target_station_id);

```

```sql
UNIQUE KEY uk_eta_latest (vehicle_id, target_station_id);
```

## 3.10 `passenger_flow_trend` 客流历史趋势表

### 用途

存储站点历史客流，用于趋势图、管理员客流表和数据概览。

### 字段设计

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `flow_record_id` | BIGINT | PK, AUTO_INCREMENT | 主键 |
| `target_type` | VARCHAR(20) | NOT NULL | 当前固定为 `station` |
| `target_id` | BIGINT | NOT NULL | 站点 ID，对应 `bus_station.station_id` |
| `bus_stop_code` | VARCHAR(30) | NULL | LTA 原始站点编号，便于追溯 |
| `record_time` | DATETIME | NOT NULL | 客流记录时间 |
| `day_type` | VARCHAR(20) | NULL | `weekday`、`weekend`、`holiday` |
| `tap_in_volume` | INT | NOT NULL, DEFAULT 0 | 上车或进入量 |
| `tap_out_volume` | INT | NOT NULL, DEFAULT 0 | 下车或离开量 |
| `total_flow` | INT | NOT NULL, DEFAULT 0 | 总客流 |
| `flow_level` | VARCHAR(20) | NULL | `low`、`medium`、`high`、`very_high` |
| `data_source` | VARCHAR(100) | NULL | 数据来源 |
| `created_at` | DATETIME | NOT NULL | 入库时间 |

### 索引

```sql
KEY idx_flow_target_time (target_type, target_id, record_time);
KEY idx_flow_record_time (record_time);
KEY idx_flow_level (flow_level);
```

## 3.11 `passenger_flow_prediction` 客流预测结果表

### 用途

预测未来某时刻的站点或路段客流。该表不能用 `load_prediction` 替代。

### 字段设计

| 字段名 | 类型 | 约束 | 说明 |

| `prediction_time` | DATETIME | NOT NULL | 本次预测生成时间 |
| `predict_time` | DATETIME | NOT NULL | 被预测的未来时间 |
| `predicted_flow` | INT | NOT NULL | 预测客流量 |
| `crowd_level` | VARCHAR(20) | NOT NULL | 拥挤等级 |
| `confidence` | DECIMAL(5,4) | NULL | 置信度 |
| `model_version` | VARCHAR(100) | NULL | 模型版本 |
| `created_at` | DATETIME | NOT NULL | 创建时间 |

### 索引

```sql
KEY idx_flow_prediction_target_time (target_type, target_id, predict_time);
KEY idx_flow_prediction_created (prediction_time);
```

# 5. 建议建立的视图

## 5.1 `v_map_station` 地图站点视图

不建议长期维护独立的 `map_station` 实体表。站点坐标来自 `bus_station`，线路列表来自 `line_station` 和 `bus_line`。

### 建议字段

```text
station_id
station_name
longitude
latitude
line_count
line_ids
line_names
service_nos
```

### 示例 SQL

```sql
CREATE VIEW v_map_station AS
SELECT
    s.station_id,
    s.station_name,
    s.longitude,
    s.latitude,
    COUNT(DISTINCT ls.line_id) AS line_count,
    GROUP_CONCAT(DISTINCT l.line_id ORDER BY l.line_id) AS line_ids,
    GROUP_CONCAT(DISTINCT l.line_name ORDER BY l.line_id) AS line_names,
    GROUP_CONCAT(DISTINCT l.service_no ORDER BY l.line_id) AS service_nos
FROM bus_station s
LEFT JOIN line_station ls ON ls.station_id = s.station_id
LEFT JOIN bus_line l ON l.line_id = ls.line_id
GROUP BY
    s.station_id,
    s.station_name,
    s.longitude,
    s.latitude;
```

# 11. 数据库实施注意事项

1. 密码只保存哈希值，禁止保存明文密码。
2. 经度和纬度统一使用 `DECIMAL(10,7)`。
3. 时间统一使用 `DATETIME`，接口层再格式化为统一字符串。
4. 所有预测表必须包含 `prediction_time`、`model_version` 和必要的置信度字段。
5. `load_prediction` 表示车内客载，不能替代站点或路段客流预测表。
6. `bus_vehicle.operation_status` 与 `predicted_load_level` 必须分开。
7. `map_station` 建议改为视图，避免与 `bus_station` 重复存储。
8. 路线推荐可以实时计算，不强制建立推荐结果表。
9. AI 助手不保存聊天历史时，AI 相关表可以暂不建立。
10. 数据更新接口写数据库时应使用事务，并同步更新 `updated_at`。
11. CSV 导入前应统一 ID 类型、枚举值、时间格式和空值规则；基础数据表导入时保留脚本生成的 `station_id`、`line_id`、`vehicle_id`，不重新自增生成。
12. 表名与后端 ORM 模型名应保持一一对应，避免同一含义出现多种命名。
