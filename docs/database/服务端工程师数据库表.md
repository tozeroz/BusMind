# 服务端工程师数据库表

## 1\.1 合并与冲突处理原则

- bus\_line、bus\_station、line\_station、bus\_vehicle、bus\_realtime 为 A/B 共用表，仅保留一份统一定义。

- bus\_line 的起终点以 start\_station\_id、end\_station\_id 为规范化关联字段；start\_station、end\_station 可作为兼容服务端 A 展示需求的可选冗余名称字段。

- 线路方向统一存放在 line\_station\.direction 中，不在 bus\_line 中重复维护，避免同一线路上下行数据冲突。

- line\_station\.distance\_to\_next 按服务端 B 的 ETA 与推荐计算需求设为必填；其他平均时间与累计距离字段可选。

- bus\_realtime 可采用“每车一条最新状态”或“保存历史轨迹”两种策略，项目组应在建表前确定。

## 1\.2 数据表总览

|**类别**|**表名**|**主要使用方**|**当前直接需要**|**主要用途**|
|---|---|---|---|---|
|**必须**|user|服务端 A|是|用户注册、登录、JWT 鉴权和个人信息|
|**必须**|user\_favorite|服务端 A|是|用户收藏常用路线或出行方案|
|**必须**|query\_history|服务端 A/B|是|记录线路、站点、推荐及 AI 助手查询历史|
|**必须**|bus\_line|A/B 共用|是|公交线路基础信息和运营信息|
|**必须**|bus\_station|A/B 共用|是|站点名称、坐标、状态及最近站点匹配|
|**必须**|line\_station|A/B 共用|是|线路站点顺序、方向、距离及候选路线计算|
|**必须**|bus\_vehicle|A/B 共用|是|车辆所属线路、容量、车型和运行状态|
|**必须**|bus\_realtime|A/B 共用|是|车辆实时位置、速度、载客量和停站状态|
|**必须**|passenger\_volume\_stop|服务端 B/模型|是|站点小时客流及客载预测特征|
|**建议**|bus\_line\_map|服务端 A|否|线路地图折线与地图视野边界|
|**建议**|eta\_prediction|服务端 B|否|ETA 预测历史、影响因素和模型版本|
|**建议**|load\_prediction|服务端 B|否|客载预测历史、体验得分和置信度|
|**建议**|route\_recommendation|服务端 B|否|路线推荐方案、分数、标签和推荐理由|



# 二、必须建立的数据表

## 2\.1 user（用户表）

用于用户注册、登录、JWT 鉴权和个人信息管理。

|**字段名**|**建议类型**|**必填**|**说明**|
|---|---|---|---|
|**user\_id**|BIGINT|是|主键，自增|
|**username**|VARCHAR\(50\)|是|登录账号，系统内唯一，建议长度 4–32 字符|
|**password\_hash**|VARCHAR\(255\)|是|bcrypt 等安全算法生成的密码哈希，不存储明文密码|
|**nickname**|VARCHAR\(32\)|否|用户昵称，默认可为空|
|**email**|VARCHAR\(100\)|否|邮箱，可选；填写时应唯一|
|**phone**|VARCHAR\(20\)|否|手机号，可选|
|**created\_at**|DATETIME|否|注册时间，默认当前时间|
|**updated\_at**|DATETIME|否|更新时间，建议自动更新|
|**last\_login\_at**|DATETIME|否|最后登录时间|



**约束与索引：**UNIQUE\(username\)；UNIQUE\(email\)。

## 2\.2 user\_favorite（用户收藏路线表）

用于用户收藏常用线路或完整出行方案。

|**字段名**|**建议类型**|**必填**|**说明**|
|---|---|---|---|
|**favorite\_id**|BIGINT|是|主键，自增|
|**user\_id**|BIGINT|是|外键关联 user\.user\_id|
|**route\_name**|VARCHAR\(100\)|否|用户自定义收藏名称|
|**start\_station\_id**|BIGINT|是|起点站编号，关联 bus\_station\.station\_id|
|**end\_station\_id**|BIGINT|是|终点站编号，关联   bus\_station\.station\_id|
|**line\_ids**|JSON|是|收藏方案涉及的线路编号列表|
|**route\_snapshot**|JSON|否|保存前端展示所需的路线快照|
|**created\_at**|DATETIME|否|收藏时间，默认当前时间|



**约束与索引：**INDEX\(user\_id\)；可按业务规则设置 UNIQUE\(user\_id, start\_station\_id, end\_station\_id, line\_ids\)。

## 2\.3 query\_history（用户查询历史表）

用于记录用户的线路、站点、推荐和 AI 助手查询行为。

|**字段名**|**建议类型**|**必填**|**说明**|
|---|---|---|---|
|**history\_id**|BIGINT|是|主键，自增|
|**user\_id**|BIGINT|是|外键关联 user\.user\_id；匿名查询场景可按项目规则允许为空|
|**query\_type**|VARCHAR\(20\)|是|查询类型：line /   station / recommend / ai|
|**query\_content**|VARCHAR\(255\)|是|查询内容；简单内容存字符串，复杂内容可存   JSON 字符串|
|**query\_time**|DATETIME|否|查询时间，默认当前时间|



**约束与索引：**INDEX\(user\_id, query\_time\)；INDEX\(query\_type\)。

## 2\.4 bus\_line（公交线路表）

用于线路列表、线路详情、运营时间、线路筛选以及推荐结果展示。

|**字段名**|**建议类型**|**必填**|**说明**|
|---|---|---|---|
|**line\_id**|BIGINT|是|主键；服务端 A/B 统一按整数   ID 使用|
|**line\_name**|VARCHAR\(100\)|是|线路名称，例如“校园 1 号线”|
|**line\_type**|VARCHAR\(20\)|是|线路类型：loop（环线）/   bidirectional（往返线）等|
|**start\_station\_id**|BIGINT|否|起点站编号，关联 bus\_station\.station\_id；作为规范化主字段|
|**end\_station\_id**|BIGINT|否|终点站编号，关联   bus\_station\.station\_id；作为规范化主字段|
|**start\_station**|VARCHAR\(100\)|否|起点站名称；兼容展示需求的可选冗余字段|
|**end\_station**|VARCHAR\(100\)|否|终点站名称；兼容展示需求的可选冗余字段|
|**first\_bus\_time**|TIME|否|首班时间，例如 07:00|
|**last\_bus\_time**|TIME|否|末班时间，例如 22:30|
|**interval\_minutes**|INT|否|发车间隔，单位：分钟|
|**total\_stations**|INT|否|线路总站数，可由   line\_station 统计后回填|
|**line\_color**|VARCHAR\(10\)|否|前端展示颜色，例如 \#FF6B6B|
|**status**|VARCHAR\(20\)|是|normal / stopped|
|**created\_at**|DATETIME|否|创建时间|
|**updated\_at**|DATETIME|否|更新时间|



**约束与索引：**INDEX\(status\)；INDEX\(line\_name\)。线路方向统一由 line\_station\.direction 表示。

## 2\.5 bus\_station（公交站点表）

用于站点查询、最近站点匹配、步行距离计算和线路站点关联。

|**字段名**|**建议类型**|**必填**|**说明**|
|---|---|---|---|
|**station\_id**|BIGINT|是|主键；服务端 A/B 统一按整数   ID 使用|
|**station\_name**|VARCHAR\(100\)|是|站点名称|
|**longitude**|DECIMAL\(10,7\)|是|经度；统一采用项目约定坐标系|
|**latitude**|DECIMAL\(10,7\)|是|纬度；统一采用项目约定坐标系|
|**station\_type**|VARCHAR\(20\)|否|普通站 / 枢纽站 / 终点站等|
|**address**|VARCHAR\(255\)|否|站点地址或位置描述|
|**status**|VARCHAR\(20\)|是|normal /   disabled|
|**created\_at**|DATETIME|否|创建时间|
|**updated\_at**|DATETIME|否|更新时间|



**约束与索引：**INDEX\(station\_name\)；INDEX\(status\)；INDEX\(longitude, latitude\)。

## 2\.6 line\_station（线路站点关联表）

用于维护线路方向与站点顺序，并支持剩余站数、剩余距离、ETA、候选路线和换乘站计算。

|**字段名**|**建议类型**|**必填**|**说明**|
|---|---|---|---|
|**id**|BIGINT|是|主键，自增|
|**line\_id**|BIGINT|是|外键关联 bus\_line\.line\_id|
|**station\_id**|BIGINT|是|外键关联   bus\_station\.station\_id|
|**direction**|TINYINT|否|0=上行，1=下行；环线可统一为 0|
|**station\_order**|INT|是|站点在线路及方向中的顺序，从 1 开始|
|**distance\_to\_next**|DECIMAL\(10,2\)|是|到下一站距离，单位：米；末站可为 0 或 NULL，需统一约定|
|**average\_travel\_minutes**|DECIMAL\(6,2\)|否|到下一站平均行驶时间，单位：分钟|
|**distance\_from\_start**|DECIMAL\(10,2\)|否|从该方向起点到本站的累计距离，单位：米|



**约束与索引：**UNIQUE\(line\_id, direction, station\_order\)；INDEX\(line\_id, direction, station\_order\)；INDEX\(station\_id\)。

## 2\.7 bus\_vehicle（车辆基础信息表）

用于校验车辆与线路关系、读取车辆容量、车型和运行状态。

|**字段名**|**建议类型**|**必填**|**说明**|
|---|---|---|---|
|**vehicle\_id**|BIGINT|是|主键|
|**line\_id**|BIGINT|是|外键关联 bus\_line\.line\_id，表示车辆当前所属线路|
|**vehicle\_no**|VARCHAR\(50\)|否|车辆编号或车牌号|
|**capacity**|INT|是|核定容量，必须大于 0|
|**vehicle\_type**|VARCHAR\(20\)|否|车型，例如大巴 / 中巴|
|**status**|VARCHAR\(20\)|是|normal / delayed / offline|
|**created\_at**|DATETIME|否|创建时间|
|**updated\_at**|DATETIME|否|更新时间|



**约束与索引：**INDEX\(line\_id\)；INDEX\(status\)；可对 vehicle\_no 设置唯一约束。

## 2\.8 bus\_realtime（车辆实时状态表）

用于车辆位置展示、ETA 计算和客载状态预测。

|**字段名**|**建议类型**|**必填**|**说明**|
|---|---|---|---|
|**id**|BIGINT|是|主键；每车只保留最新状态时也可采用   vehicle\_id 唯一覆盖更新|
|**vehicle\_id**|BIGINT|是|外键关联 bus\_vehicle\.vehicle\_id|
|**line\_id**|BIGINT|是|外键关联   bus\_line\.line\_id|
|**longitude**|DECIMAL\(10,7\)|是|车辆当前经度|
|**latitude**|DECIMAL\(10,7\)|是|车辆当前纬度|
|**current\_station\_id**|BIGINT|否|当前或最近站点编号|
|**next\_station\_id**|BIGINT|否|下一站编号|
|**progress**|DECIMAL\(5,4\)|否|当前站至下一站的进度，范围 0–1|
|**speed\_kph**|DECIMAL\(6,2\)|是|当前速度，单位 km/h|
|**onboard\_count**|INT|是|当前车上人数，必须大于等于 0|
|**status**|VARCHAR\(20\)|是|normal / delayed   / offline|
|**update\_time**|DATETIME|是|本条实时状态更新时间|



**约束与索引：**最新状态模式：UNIQUE\(vehicle\_id\) 并执行 upsert；历史轨迹模式：INDEX\(vehicle\_id, update\_time\)，数据量大时按日期分区。

## 2\.9 passenger\_volume\_stop（站点小时级客流表）

用于客载预测的站点客流特征；规则降级至少依赖 flow\_level，正式模型可使用上下车客流、天气和日期类型。

|**字段名**|**建议类型**|**必填**|**说明**|
|---|---|---|---|
|**id**|BIGINT|是|主键，自增|
|**station\_id**|BIGINT|是|外键关联 bus\_station\.station\_id|
|**stat\_date**|DATE|是|统计日期|
|**day\_type**|VARCHAR\(20\)|是|weekday / weekend / holiday|
|**time\_per\_hour**|TINYINT|是|小时，范围 0–23|
|**tap\_in\_volume**|INT|是|该站点该小时上车人数|
|**tap\_out\_volume**|INT|是|该站点该小时下车人数|
|**flow\_level**|VARCHAR\(20\)|否|low / medium / high；可根据客流量规则计算|
|**weather**|VARCHAR\(30\)|否|clear / rain /   snow / storm 等|
|**created\_at**|DATETIME|否|创建时间|



**约束与索引：**UNIQUE\(station\_id, stat\_date, time\_per\_hour\)；INDEX\(station\_id, stat\_date, time\_per\_hour\)。

# 三、建议建立的辅助、预测与推荐结果表

**说明：**以下表不属于当前接口正常运行的硬性依赖，但建立后可以保存预测与推荐历史、支持模型评估、问题追踪和前端地图展示。

|**bounds\_max\_lon**|DECIMAL\(10,7\)|否|地图视野最大经度|

# 四、接口与模型输入输出数据协议

## 4\.1 用户注册输入示例

\{
  "username": "xhh",
  "password": "12345678",
  "nickname": "会惠"
\}

**说明：**接口接收 password 明文仅用于本次请求；服务端写入 user 表前必须生成 password\_hash，禁止将明文密码保存到数据库。

## 4\.2 ETA 模型输入

服务端 B 向 ETA 模型传入以下特征字典。数据处理工程师提供的函数应直接接收该结构，或由适配器完成字段映射。

\{
  "vehicle\_id": 101,
  "line\_id": 1,
  "target\_station\_id": 12,
  "distance\_to\_stop": 1850\.0,
  "remaining\_stop\_count": 4,
  "speed\_kph": 25\.0,
  "hour": 8,
  "day\_type": "weekday",
  "is\_peak": true
\}

## 4\.3 ETA 模型最低输出

\{
  "predicted\_eta\_minutes": 7\.4,
  "model\_version": "eta\_random\_forest\_v1"
\}

模型也可以仅返回数值 7\.4。模型导入失败或执行异常时，当前服务端 B 代码可降级使用 eta\_rule\_v1。

## 4\.4 客载状态模型输入

\{
  "line\_id": 1,
  "station\_id": 3,
  "vehicle\_id": 101,
  "hour": 8,
  "day\_type": "weekday",
  "is\_peak": true,
  "station\_flow\_level": "high",
  "weather": "rain",
  "current\_onboard\_count": 40,
  "capacity": 60
\}

## 4\.5 客载状态模型建议输出

\{
  "predicted\_onboard\_count": 46,
  "predicted\_load\_rate": 0\.77,
  "predicted\_load\_level": "SDA",
  "confidence": 0\.84,
  "model\_version": "load\_random\_forest\_v1"
\}

|**模型标签**|**接口枚举**|**中文含义**|
|---|---|---|
|**SEA**|seats\_available|有座位，较宽松|
|**SDA**|standing\_available|可站立，适中|
|**LSD**|limited\_standing|站立空间有限，较拥挤|



# 五、数据库约束、索引与字段规范

|**项目**|**统一规范**|
|---|---|
|**命名规则**|数据库表名与字段名统一使用小写下划线   snake\_case。|
|**ID 类型**|user\_id、line\_id、station\_id、vehicle\_id 等业务主键统一使用   BIGINT。|
|**时间字段**|数据库统一使用 DATETIME；接口输出统一为   ISO 8601 格式。|
|**经纬度**|固定使用 longitude、latitude，并在项目内统一坐标系；不得混用 WGS84、GCJ\-02   等坐标。|
|**速度单位**|固定使用 speed\_kph，单位   km/h，避免使用含义不明确的 speed。|
|**距离单位**|distance\_to\_next、distance\_from\_start、distance\_meters   统一使用米。|
|**比例范围**|progress、predicted\_load\_rate、confidence   的合法范围均为 0–1。|
|**得分范围**|load\_score、experience\_score 的合法范围为 0–100。|
|**JSON 字段**|line\_ids、segments、polyline   等字段由应用层校验结构，避免写入无法解析的字符串。|
|**枚举字段**|status、day\_type、flow\_level、predicted\_load\_level 等值应由后端枚举统一管理。|
|**外键策略**|核心主数据表建议启用外键；高频实时表和预测历史表可结合性能需求决定是否使用物理外键。|
|**删除策略**|线路、站点和车辆优先通过 status 软停用，避免直接删除导致历史记录失去关联。|
