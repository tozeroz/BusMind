# BusMind 数据流转链路实现

> 当前口径：后端线上数据链路与模型离线训练链路分离。后端负责实时数据采集、缓存、入库和候选路线特征组装；模型只消费后端传入的结构化路线特征，离线训练只消费 `algorithm/dataset/recommendation/v1` 下的数据集。

## 1. 总体链路

```text
LTA DataMall
    ↓
data/raw/lta
    ↓
data/processed/process_scripts/build_lta_processed_data.py
    ↓
data/processed/*.csv
    ↓
database/import/import_processed_to_mysql.py
    ↓
MySQL + MemoryCacheProvider
    ↓
后端 ETA / Load / Recommend 服务
    ↓
algorithm.model.predictor.predict_recommendation
    ↓
前端展示 + AI 只解释结构化推荐结果
```

模型离线训练链路单独收口：

```text
data/raw/lta 或 data/processed
    ↓
algorithm/dataset/build_features.py
    ↓
algorithm/dataset/recommendation/v1/features.csv
    ↓
algorithm/dataset/build_labels.py
    ↓
algorithm/dataset/recommendation/v1/pseudo_labels.csv
    ↓
algorithm/model/train_ranker.py
    ↓
algorithm/model/artifacts/route_scorer_v1.json
```

## 2. 数据目录职责

| 目录 | 职责 | 是否模型直接依赖 |
|---|---|---|
| `data/raw/lta` | 保存 LTA 原始 JSON / ZIP / JSONL 快照 | 间接依赖 |
| `data/processed` | 保存清洗后的项目通用 CSV | 间接依赖 |
| `database` | MySQL schema 与 CSV 导入脚本 | 否 |
| `backend` | 线上实时查询、缓存、入库、候选路线组装 | 线上推理时组装 payload |
| `algorithm/dataset` | 构建模型专用训练数据集 | 是 |
| `algorithm/model` | 推荐评分模型、训练脚本和模型产物 | 是 |

## 3. 后端实时数据链路

后端实时链路已经按 LTA 官方 API 接入：

```text
LtaDataMallClient
    ↓
LtaCollectorService
    ↓
MemoryCacheProvider
    ↓
CacheSyncService
    ↓
MySQL 状态表
```

当前已覆盖：

| 数据 | LTA 接口 | 缓存 key | 数据库表 |
|---|---|---|---|
| Bus Arrival | `v3/BusArrival` | `bus_arrival:{stop_code}`、`bus_arrival:{stop_code}:{service_no}` | `lta_bus_arrival`、`bus_eta_status`、`bus_load_status` |
| Traffic Speed Bands | `v4/TrafficSpeedBands` | `traffic_speed_bands:latest`、`traffic_speed_bands:{link_id}` | `traffic_speed_bands` |

未完成项：

1. Redis Provider 尚未落地，目前仍是进程内内存缓存。
2. APScheduler 自动刷新仍未完全接入，当前主要依赖管理接口触发。
3. 热门站点刷新策略、限流、重试和缓存失效策略仍需补齐。

## 4. 推荐模型数据链路

模型不调用 LTA、不查 MySQL、不访问缓存。模型接收后端已组装好的候选路线特征。

线上推理：

```text
RecommendationService
    ↓
候选路线 + ETA + Load + 客流 + 路况 + 可靠性
    ↓
ModelScoringRequest
    ↓
algorithm.model.predictor.predict_recommendation(payload)
    ↓
五维分数 + recommend_score
```

离线训练：

```text
algorithm/dataset/build_features.py
algorithm/dataset/build_labels.py
algorithm/model/train_ranker.py
```

## 5. 当前模型输入字段

每条路线固定 12 个数值特征：

```text
eta_minutes
ride_time_minutes
walk_time_minutes
walk_distance_meters
transfer_count
load_score
history_flow_score
congestion_score
data_freshness_seconds
monitored_score
completeness_score
avg_service_frequency_minutes
```

辅助元数据：

```text
route_id
service_nos
load_code
feature_sources
degraded_fields
```

## 6. 当前结论

数据流转边界已经固定：

- `data` 负责离线原始和清洗数据。
- `backend` 负责线上实时数据与候选路线特征组装。
- `algorithm/dataset` 负责模型训练数据集。
- `algorithm/model` 负责模型训练与推理。

后续不要再让模型训练脚本直接混用后端接口、ps1 脚本和散落 CSV。模型训练统一从 `algorithm/dataset/recommendation/v1` 读数据，线上推理统一从后端传入 payload。
