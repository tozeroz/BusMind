# 智行公交项目目录说明

## 一、项目目录总览

本项目采用前后端分离结构，前端使用 **Vue3 + Element Plus + ECharts + 地图 API**，后端使用 **FastAPI**，数据库使用 **MySQL**，算法部分使用 **Python + Pandas + Scikit-learn / XGBoost**，并预留大模型助手接口。完整目录结构如下：

```text
BusMind/
├── frontend/                         # 前端项目目录，存放 Vue3 页面、组件、路由、接口请求等
│   ├── public/                       # 前端静态资源目录
│   ├── src/                          # 前端核心源码目录
│   │   ├── api/                      # Axios 接口请求封装
│   │   ├── assets/                   # 图片、图标、字体等资源
│   │   ├── components/               # 公共组件目录
```

## 二、前端目录说明

前端所有代码放在：

```text
frontend/
```

## 五、算法目录说明

| 目录 | 说明 |

### 5.1 ETA 与客载状态预测目录

ETA 预测和客载状态预测是推荐算法的上游，建议拆得更清楚：

```text
algorithm/eta_prediction/
├── features/          # 构造 ETA 特征
├── baseline/          # 平均值 baseline、历史均值 baseline
├── models/            # 训练和预测 ETA 回归模型
└── evaluation/        # MAE、RMSE 评价

algorithm/load_prediction/
├── features/          # 构造 Load 特征
├── baseline/          # 多数类 baseline、历史众数 baseline
├── models/            # 训练和预测 Load 分类模型
└── evaluation/        # Accuracy、Macro-F1、混淆矩阵评价
```

开发顺序建议：

```text
先整理 LTA 风格字段 → 构造训练集和测试集 → 建立 baseline → 训练模型 → 接入推荐模块
```

```text
data/
```

| 目录 | 说明 |
|---|---|

```text
simulator/
```

该目录用于在没有真实公交接口时，模拟公交车辆位置、客流变化、ETA 标签和客载状态标签。

| 目录 | 说明 |
|---|---|
| `simulator/vehicle_position/` | 模拟车辆在线路上移动 |
| `simulator/passenger_flow/` | 模拟早高峰、晚高峰、雨天等客流变化和 Load 标签 |
| `simulator/scheduler/` | 定时任务，比如每 5 秒更新车辆位置 |

该目录的作用是让项目在没有真实数据接口的情况下也能完整演示。

## 八、大模型助手目录说明

大模型相关代码放在：

```text
llm/
```

该目录主要用于管理 Prompt、调用 DeepSeek / 通义千问 / OpenAI API，并生成自然语言出行建议。

| 目录 | 说明 |
|---|---|
| `llm/prompts/` | 大模型 Prompt 模板 |
| `llm/providers/` | 不同大模型服务商接口封装 |
| `llm/providers/deepseek/` | DeepSeek API 封装 |
| `llm/providers/tongyi/` | 通义千问 API 封装 |
| `llm/providers/openai/` | OpenAI API 封装 |
| `llm/context_builders/` | 把线路、预测 ETA、客载状态、推荐结果整理成上下文 |

注意：  
大模型不应该直接负责训练或预测 ETA、客载状态。正确流程应该是：

```text
后端 / 算法模块先计算结果
        ↓
整理成结构化上下文
        ↓
交给大模型生成解释文本
```

## 十五、核心业务数据流

系统主要业务流程如下：

```text
用户输入起点和终点
        ↓
前端通过 Axios 请求后端推荐接口
        ↓
后端查询线路、站点、车辆、历史客流和候选方案数据
        ↓
调用 ETA 预测模块得到 predicted_eta_minutes
        ↓
调用客载状态预测模块得到 predicted_load 和 load_score
        ↓
调用推荐模块计算出行体验分数
        ↓
可选：调用大模型生成推荐解释
        ↓
后端返回推荐方案
        ↓
前端用地图、卡片、图表展示结果
```

## 十六、各模块之间的关系

| 模块 | 依赖模块 | 输出结果 |
|---|---|---|
| 前端模块 | 后端接口 | 页面展示 |
| 后端模块 | 数据库、算法、大模型 | RESTful API 数据 |
| 数据库模块 | 无 | 线路、站点、车辆、客流数据 |
| ETA 模块 | 线路、站点、车辆状态、时间、天气、历史 ETA | `predicted_eta_minutes` |
| 客载状态预测模块 | 历史 Load、站点客流、OD 客流、时间、天气 | `predicted_load`、`load_score` |
| 推荐模块 | 预测 ETA、客载状态、换乘次数、步行距离 | 出行体验推荐方案 |
| 大模型模块 | 推荐结果、路线数据、ETA 和客载状态 | 自然语言解释 |

## 十七、总结

本目录结构按照“前端展示、后端接口、数据库存储、算法计算、大模型解释、文档与测试分离”的原则设计，适合本项目的课程开发和小组协作。

整体结构的核心思想是：

```text
frontend 负责页面展示
backend 负责接口和业务调度
database 负责数据存储
algorithm 负责 ETA 预测、客载状态预测和推荐计算
simulator 负责生成模拟车辆、客流、ETA 标签和 Load 标签
llm 负责大模型解释和自然语言建议
docs 负责项目文档和答辩材料
tests 负责测试验证
deployment 负责后期部署
```

这样拆分后，每个成员可以在自己的目录内独立开发，最后通过 RESTful API 和 Axios 完成前后端联调。
