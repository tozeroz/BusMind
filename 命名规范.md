# 智行公交项目命名规范

> 本规范只约束项目中最容易混乱的命名：Python 包和代码、前端文件、数据库、API 路径、SQL/CSV 文件、空目录占位、MD、YAML、JSON 文件

# 一、空目录占位

空目录统一使用 `.gitkeep` 占位

推荐：

```text
data/raw/.gitkeep
algorithm/crowd_prediction/.gitkeep
backend/app/services/.gitkeep
```

不使用：

```text
目录.txt
readme.txt
placeholder.txt
```

# 二、目录命名边界

项目根目录或普通外层工程目录可以使用小写短横线

```text
bus-mind
project-docs
demo-assets
```

项目内部如果是 Python 包、脚本模块、数据处理模块，统一使用小写下划线

```text
crowd_prediction
data_processing
model_files
passenger_flow
vehicle_position
context_builders
import_data
train_model
```

前端路由目录和 API 路径可以使用小写短横线

```text
ai-assistant
crowd-prediction
recommend-routes
```

# 三、Python 项目包命名

Python 包目录需要考虑后续 `import`，因此统一使用**小写下划线**命名

推荐：

```text
data_processing
crowd_prediction
rule_model
ml_model
model_files
eta_service
recommend_service
```

不推荐：

```text
data-processing
CrowdPrediction
crowdPrediction
拥挤预测
```

判断原则：

```text
只要后续可能被 Python import，就使用 xx_xx
```

Python 文件同样使用小写下划线：

```text
generate_mock_data.py
train_crowd_model.py
predict_crowd.py
calculate_eta.py
recommend_route.py
```

# 四、Python 程序内部命名

## (一) 变量和函数

变量和函数使用小写下划线

```python
crowd_rate = onboard_count / capacity
eta_minutes = 8

def calculate_eta_minutes():
    pass

def predict_crowd_level():
    pass
```

## (二) 类名

类名使用 `PascalCase`

```python
class CrowdPredictor:
    pass

class EtaCalculator:
    pass

class RouteRecommender:
    pass
```

## (三) 常量

常量使用大写下划线

```python
MAX_CAPACITY = 60
DEFAULT_WEATHER = "sunny"
PEAK_HOURS = [7, 8, 17, 18]
```

```text

```

不推荐：

```text

BusStation.csv
公交站点.csv
```

# 九、API 路径命名

API 路径使用小写短横线

推荐：

```text
/api/v1/users/login
/api/v1/users/register
/api/v1/bus-lines
/api/v1/bus-stations
/api/v1/bus-vehicles/realtime
/api/v1/eta
/api/v1/crowd-prediction
/api/v1/recommend-routes
/api/v1/ai/travel
```

不推荐：

```text
/api/v1/bus_lines
/api/v1/getLineList
/api/v1/公交线路
```

接口返回字段使用小写下划线，尽量和数据库字段、CSV 字段保持一致

```json
{
  "line_id": "L001",
  "station_id": "S001",
  "eta_minutes": 8,
  "crowd_level": "comfortable"
}
```

# 十、MD、YAML、JSON 文件命名

## (一) MD 文件

Markdown 文件统一使用中文或小写短横线，名称要清楚表达内容

推荐：

```text
directory-guide.md
naming-rules.md
data-collection-guide.md
database-design.md
api-guide.md
daily-problem-list.md
```

不推荐：

```text
doc1.md
new.md
```

特别的，类似 `README.md` 这种特殊的，名称要全大写

## (二) YAML 文件

YAML 文件通常用于配置，统一使用小写短横线

推荐：

```text
app-config.yaml
database-config.yaml
docker-compose.yaml
api-config.yaml
```

不推荐：

```text
DatabaseConfig.yaml
配置.yaml
app_config.yaml
```

## (三) JSON 文件

JSON 文件通常用于配置、接口样例、模拟数据

- 程序读取、数据类  JSON 统一使用小写下划线，例如：`mock_bus_line.json`、`api_response_example.json`
- 前端静态配置、页面资源类 JSON 统一使用小写短横线，例如：`route-config.json`
