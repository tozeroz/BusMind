# 项目组织与职责分工计划

## 项目组织与职责分工

## 项目经理

### 核心定位

项目经理负责项目整体推进、需求整理、进度协调、文档汇总和答辩准备，同时协助完成部分后端基础工作和数据整理工作，保证项目按计划推进。

### 主要职责

| 职责方向 | 具体任务                                                  |
| -------- | --------------------------------------------------------- |
| 项目统筹 | 制定开发计划，跟踪每日进度，协调成员任务                  |
| 需求分析 | 明确项目目标、用户角色、核心功能和功能边界                |
| 文档整理 | 整理项目计划书、需求文档、接口说明、项目报告              |
| 答辩准备 | 制作 PPT，编写答辩稿，设计演示流程                        |
| 后端协助 | 协助搭建 FastAPI 基础结构，整理接口文档，协助简单接口开发 |
| 数据协助 | 协助整理公交线路、站点、车辆和历史客流模拟数据            |
| AI 设计  | 设计 AI 出行助手的提示词、推荐解释文案和演示问题          |

### 交付物

```
项目进度计划
需求分析文档
系统功能模块图
接口文档初稿
AI 助手提示词设计
中期汇报 PPT
最终答辩 PPT
答辩稿
演示脚本
项目总结文档
```

## 服务端工程师 A

### 核心定位

服务端工程师 A 主要负责基础业务接口，任务以简单、稳定、容易联调为主，不承担复杂算法和 AI 接入。

### 主要职责

| 职责方向     | 具体任务                                   |
| ------------ | ------------------------------------------ |
| 基础接口开发 | 配合完成 FastAPI 后端基础项目结构          |
| 线路查询     | 实现查询所有公交线路接口                   |
| 线路详情     | 实现查询某条线路详细信息接口               |
| 站点查询     | 实现查询线路站点列表接口                   |
| 车辆状态查询 | 实现车辆当前位置、下一站、运行状态查询接口 |
| 数据库对接   | 配合 MySQL 读取线路、站点、车辆基础数据    |
| 前端联调     | 配合客户端完成线路查询和公交详情页面联调   |

### 交付物

```
公交线路查询接口
线路详情接口
站点列表接口
车辆状态查询接口
基础业务接口说明
接口联调记录
```

## 服务端工程师 B

### 核心定位

服务端工程师 B 负责项目中的智能功能接口，包括 ETA 计算、拥挤预测调用、舒适推荐和 AI 出行助手接口，是后端智能模块的主要负责人。

### 主要职责

| 职责方向     | 具体任务                                           |
| ------------ | -------------------------------------------------- |
| ETA 接口     | 根据车辆位置、站间距离和时段因素计算预计到站时间   |
| 拥挤预测接口 | 调用数据处理工程师提供的预测结果或模型             |
| 舒适推荐接口 | 综合 ETA、拥挤程度、步行距离和换乘次数生成推荐方案 |
| AI 助手接口  | 接入大模型接口，生成自然语言出行建议               |
| 推荐解释     | 返回推荐理由，说明为什么推荐某条线路               |
| 接口测试     | 完成核心接口测试和异常情况处理                     |

### 交付物

```
ETA 到站时间接口
拥挤程度预测接口
舒适路线推荐接口
推荐理由解释接口
AI 出行助手接口
接口测试记录
```

## 客户端工程师

### 核心定位

客户端工程师负责 Vue3 前端页面开发和前后端联调，重点完成乘客端主要功能页面，保证系统能够直观展示和稳定演示。

### 主要职责

| 职责方向 | 具体任务                                |
| -------- | --------------------------------------- |
| 前端框架 | 搭建 Vue3 项目结构，配置路由和基础布局  |
| 用户页面 | 完成登录注册页面                        |
| 查询页面 | 完成首页、起终点输入、线路查询页面      |
| 详情页面 | 展示站点列表、车辆状态、ETA、拥挤程度   |
| 推荐页面 | 展示最快方案、最舒适方案、综合推荐方案  |
| AI 页面  | 完成 AI 出行助手聊天式页面              |
| 图表展示 | 使用 ECharts 展示历史拥挤趋势或客流统计 |
| 接口联调 | 调用后端接口，完成页面数据展示          |

### 交付物

```
登录注册页面
系统首页
线路查询页面
公交详情页面
舒适推荐页面
AI 出行助手页面
历史趋势图表页面
前端联调记录
```

## 数据处理工程师

### 核心定位

数据处理工程师负责项目的数据支撑和预测模型，包括数据库表设计、模拟数据生成、历史客流数据处理和拥挤程度预测。

### 主要职责

| 职责方向   | 具体任务                                           |
| ---------- | -------------------------------------------------- |
| 数据库设计 | 设计用户、线路、站点、车辆、客流、预测结果等数据表 |
| 基础数据   | 整理公交线路、站点、站点顺序、车辆信息             |
| 模拟数据   | 生成车辆运行数据和历史客流数据                     |
| 数据预处理 | 清洗数据，构造时段、天气、站点热度等特征           |
| 拥挤预测   | 实现规则模型，若时间允许加入机器学习模型           |
| 图表数据   | 为前端提供客流趋势、站点热度、拥挤分布数据         |
| 算法说明   | 编写数据来源、预测逻辑和评价指标说明               |

### 交付物

```
数据库表结构
数据库初始化 SQL
公交线路数据
公交站点数据
车辆模拟数据
历史客流数据
拥挤预测规则或模型
图表展示数据
算法说明文档
```

# 最终分工总结

本项目组共 5 人，设项目经理 1 名、服务端工程师 2 名、客户端工程师 1 名、数据处理工程师 1 名。项目经理负责需求分析、进度管理、文档汇总、答辩组织，并协助部分后端基础工作和数据整理；服务端工程师 A 负责线路、站点、车辆等基础业务接口；服务端工程师 B 负责 ETA 计算、拥挤预测接口、舒适推荐和 AI 出行助手；客户端工程师负责 Vue3 前端页面开发和接口联调；数据处理工程师负责数据库设计、模拟数据生成、拥挤预测模型和图表数据支持。项目按照十天开发节奏推进，依次完成立项、需求分析、数据准备、架构搭建、功能开发、算法对接、前后端联调、功能完善、答辩准备和最终展示。
